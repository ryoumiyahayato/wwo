# 性能预算

## 当前状态

2026-07-19 报告中的 Alpha 数字只适用于洛岚—维斯塔 10×8 架空网格技术夹具。该网格已经被撤销为正式世界权威，因此以下内容不得继续表述为“完整 Alpha 性能验收通过”：

- 80 个正交网格单元；
- 两国八区四城；
- 基于 `AlphaWorldService` 网格批处理的 30 日、365 日和三年结果；
- `AlphaMapCanvas` 正交格子窗口 FPS；
- 由架空预设建立的存档体积和对象数量。

这些数字仍可作为旧夹具的回归参考，但不能代表正式世界地图、真实地区数据和后续生活经济的预算。

## 正式地图性能边界

正式地图使用：

- `data/world_map/*` 的国家、行政几何、城市、港口和交通数据；
- `scripts/world_map/*` 的离线几何缓存、Robinson 投影、LOD、视口裁剪、空间索引、批绘层和 Dirty Flag；
- 玩家所在地区的详细模拟、关联地区的简化模拟和远方世界的聚合模拟。

世界地图不以“64—96 个正式地图单元”为产品规模上限。旧 80 单元只能作为战争控制和性能夹具。

## 更新预算

- 业务逻辑不得在 `_process()` 或 `_physics_process()` 中全量遍历世界。
- UI 只在数据变化、镜头稳定或有限刷新频率下更新。
- 地图拖动和缩放使用已有批绘、LOD、视口裁剪、空间索引和 Dirty Flag，不为国家、地区、城市或人物逐个创建可见节点。
- 每小时不得扫描全部国家、地区、人物、企业和组织。
- 活跃人物、详细企业和当前地区使用索引与到期队列；背景对象按日、周或月聚合。
- 稳定历史必须有界，但不得通过删除仍有法律、资产、合同、债务、职位或知识意义的正式事实伪造性能。
- 正常游戏不应持续出现超过 100 ms 的主线程停顿；8 倍速不得长期冻结 UI。

## 重新测量要求

正式 Alpha 只有在以下条件成立后才能建立新基线：

1. 普通启动使用正式世界地图。
2. 国家、地区、城市、地点和交通只有一套权威。
3. 经济、劳动、企业、组织和政策引用正式地图 ID。
4. 普通 UI 能运行主要生命周期。
5. 长跑保持正式经济、AI、地区与政治系统开启。
6. 测试记录对象数量、事件数量、存档大小、峰值内存和最长单小时耗时。

每个性能项目至少运行三次并报告最小值、最大值、中位数和平均值。不能只选择最快一次。

## 目标指标

在 Windows 11 x86-64、Godot `4.6.3.stable.official.7d41c59c4`、Compatibility 下，正式 Alpha 的初始目标为：

- 1280×720 普通窗口平均不低于 30 FPS；
- 持续交互时无频繁超过 100 ms 的主线程停顿；
- 30 日无人干预模拟目标不超过 5 秒；
- 365 日无人干预模拟目标不超过 45 秒；
- 三年无人干预模拟目标不超过 150 秒；
- 单个游戏小时最坏处理时间目标不超过 250 ms；
- 存档目标不超过 5 秒；
- 载入、校验与恢复目标不超过 8 秒。

这些是目标，不得通过关闭正式模拟、减少正式地区或删除生命周期来伪造通过。未达到目标时，应报告真实值、瓶颈和优化入口。

## 仍有效的历史参考

以下历史结果仅用于回归比较：

- V2.2/V2.3 正式世界地图和生活循环的既有测试；
- `tools/run_validation.ps1` 的 Headless 导入与错误日志扫描；
- 旧 UI 和 V2.3 的窗口、导出、存档与一年模拟基线；
- 2026-07-19 架空网格 Alpha 的服务压力测试。

任何模拟公式、地图权威、AI、存档或人物结构变化后都必须重跑。历史结果不得自动继承为当前提交通过证据。

## 当前结论

- 正式世界地图入口已经恢复。
- 架空网格 Alpha 性能数据降级为技术夹具记录。
- 当前没有“完整 Alpha 性能验收通过”的有效结论。
- 待 Alpha 系统迁移到正式地图后，按本文件重新建立基线。
