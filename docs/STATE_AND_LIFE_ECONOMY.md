# 人物状态与生活经济

状态：设计冻结 v1。本文替代“单一财富、单一声望、就业布尔值”的目标设计；现有字段在迁移前仍只是技术原型。

## 1. 通用状态规则

每个核心状态必须具有稳定字段、来源明细、结算周期、作用范围、AI 用途、UI 摘要和存档归属。数值变化统一产生 `CausalRecord`，禁止 UI 通过比较前后快照猜原因。

所有阈值和系数数据驱动。范围钳制发生在领域服务，不在 UI。

## 2. 健康

```text
health: 0..100
health_conditions[]
health_trend
last_monthly_health_settlement
```

含义：长期身体状态。来源包括年龄、疾病、伤病、营养、住房、医疗、长期疲劳、长期压力、危险工作和战争环境。

结构公式：

```text
月健康变化 = 基础年龄趋势 + 营养与住房 + 医疗
             - 疾病伤病 - 长期疲劳负荷 - 长期压力负荷 - 环境风险
```

每日只处理短期病情和特殊伤害；每月汇总长期变化。健康影响行动效率、疲劳恢复、疾病风险、工作资格、军事资格和死亡风险。AI 在健康危险时提高医疗、休整和安全优先级。

UI 第一层显示整数、趋势和最主要原因；详情显示本月来源。其他人物只显示已知估计。

## 3. 疲劳

```text
fatigue: 0..100
fatigue_trend
overload_days
```

含义：短期身体和精神消耗。增加来源为工作、学习、通勤、旅行、战斗、照料和休息不足；降低来源为自动基础恢复、休整、健康、住房和生活水平。

```text
日疲劳变化 = 义务负荷 + 计划负荷 + 旅行/战斗负荷
             - 基础恢复 × 健康修正 × 住房修正 × 生活水平修正
```

高疲劳降低工作与学习效率、提高失误和压力；持续高疲劳进入月度健康损害。什么都不做自动构成基础休息。

## 4. 压力

```text
stress: 0..100
stress_trend
stressors[]
supports[]
```

增加来源：失业、债务、现金缺口、工作失败、超负荷、关系冲突、监视、拘留、政治风险、战争和家庭危机。降低来源：稳定收入、现金储备、恢复、安全环境、良好关系、工作成功和社会支持。

```text
周压力变化 = 经济不安全 + 义务冲突 + 风险事件 + 关系冲突
             - 收入稳定 - 社会支持 - 安全与恢复
```

压力降低恢复与工作表现，提高关系冲突和部分事件风险；长期高压影响健康。AI 把主要压力源加入目标评分，不能只选择“休息”而忽略债务或失业根因。

## 5. 财务账户

人物财务拆分为：

```text
PersonalFinance
  cash
  asset_ids[]
  debt_ids[]
  monthly_income_total
  monthly_expense_total
  available_credit
  net_worth
  credit_record
  last_monthly_statement_id
```

- 现金：可立即支付的流动余额，可为零但不自动为负。
- 资产：住房、土地、店铺、设备、证券、企业权益等有所有者和估值。
- 债务：债权人、余额、利率、到期、担保、拖欠状态。
- 月度收入/支出：派生摘要，不是独立生成财富的来源。
- 信用额度：由稳定收入、资产、职业、组织身份、还款记录和行业评价决定。
- 净资产：资产估值 + 应收 - 债务，不等于现金。

现金不足时，只有在信用额度和借款条件允许时才形成债务。债务利息从债务人流向债权人；违约可能导致资产处置、信用下降、生活水平下降和压力上升。

## 6. 资产、债务与资金守恒

```text
Asset
  asset_id, owner_ids[], type, location_id, quantity, book_value,
  market_value_estimate, income_rights, maintenance_cost, encumbrances[]

DebtContract
  debt_id, borrower_id, lender_id, principal, interest_rule,
  payment_schedule, collateral_asset_ids[], arrears, status
```

每一笔财富变化必须记录付款方、收款方、金额、原因和时间。基础工作工资来自雇主；消费支付给企业；税收进入政府；贷款由银行或债权人提供；破产形成坏账和资产转移。

## 7. 月度财务循环

```text
工资 + 职位津贴 + 企业利润/分红 + 资产收益 + 转移收入
- 住户贡献 - 个人消费 - 税费 - 债务利息/本金 - 资产维护
= 月度现金变化
```

结算顺序固定为：确认合同收入 → 企业/机构实际支付能力 → 税费与强制扣款 → 住户贡献与生活消费 → 债务偿还 → 可选支出。欠薪和未支付项目形成明确应收或违约，不能照常生成现金。

## 8. 就业合同

就业不是布尔值，也不要求玩家每天点击工作。

```text
EmploymentContract
  contract_id
  employee_id
  employer_organization_id
  workplace_id
  occupation_id
  position_id
  base_wage_rule
  hours_per_week
  work_intensity
  stability
  required_skills[]
  promotion_path_ids[]
  performance
  start_hour
  end_hour?
  termination_terms
  arrears
```

合同自动产生工作义务和基础工资。主动“投入工作”是个人计划或次要活动，用于提高表现、奖金、能力、晋升机会和同事关系，同时增加疲劳；它不是获得全部基础工资的唯一方式。

企业现金不足可能欠薪、减产、裁员、借款或破产。失业打开求职、临时工作、救济、搬迁、转行和家庭互助，而不是结束游戏。

## 9. 工作表现与资格

表现按周更新，读取技能、出勤、疲劳、压力、工作条件和近期投入。表现影响奖金、合同稳定、专业评价和晋升候选，但单周失败不永久锁死职业。

资格是稳定记录，例如证书、学徒完成、军衔要求或法定执照；技能水平与资格分开。

## 10. 生活水平

生活水平由目标档位和实际满足度组成：

```text
LivingStandardState
  target_tier
  affordability_ratio
  fulfillment_ratio
  consecutive_shortfall_months
```

它决定住户消费篮子，并影响健康恢复、疲劳恢复、压力、社交活动与有限的地方评价。提高档位是持续支出承诺，不是一次性购买永久增益。

## 11. 声誉与知名度

废除单一全球声望。底层至少保存：

```text
ReputationRecord
  subject_character_id
  audience_type: locality | profession | organization | political_group | country_public
  audience_id
  visibility: 0..100
  evaluation: -100..100
  trust: -100..100 (仅组织/明确群体可用)
  scandal: 0..100
  last_changed_hour
  causal_record_ids[]
```

- 地方知名度/评价只作用于具体地方。
- 职业知名度/评价只作用于行业或职业。
- 组织信任只作用于具体组织。
- 政治群体评价按利益群体或派系存储。
- 国家公众知名度用于真正全国公开人物。
- 丑闻可以按受众传播，不是全球瞬时同步。

UI 可给出“本地小有名气、工人中评价良好、企业界评价较差”等摘要。其他人物的数值受有限信息系统约束。

## 13. AI 使用优先级

AI 使用状态时先判断硬危机：健康危险、无住处、消费缺口、即将违约、失去收入。随后评估稳定收入、家庭安全、职业发展、资产、社会和制度目标。AI 目标评分读取趋势和原因，不只读取一个瞬时值。

## 14. UI 与因果详情

人物中心第一层显示健康、疲劳、压力、现金、月度结余、净资产、就业和本地/职业评价摘要。点击后显示原因账本，例如工资、住房、利息、行动投入和净变化。

正式 UI 不显示其他人物精确现金、健康、技能和完整评价；只显示知识系统允许的估计。

## 15. 存档边界

首版实施时保存权威账户、合同、资产、债务、住户成员关系、当前状态与必要原因记录；月度摘要可重建但最近玩家可见账本需持久化。迁移必须新增存档版本，不得在本设计阶段修改现有版本 1。

## 16. 首版与保留接口

首版必须：三项身体状态、个人/住户现金、基础资产债务、就业合同、工资来源、生活水平、地方/职业/组织评价和月度账本。

只保留接口：复杂证券、跨币种、保险、完整银行监管、遗产税、复杂公司股权、全国舆论分群和长期医疗专科。
