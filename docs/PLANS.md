## 复杂任务模板

每个复杂任务在编码前记录：

1. 目标与所属里程碑。
2. 明确包含与不包含的范围。
3. 现有代码、数据和文档检查结果。
4. 最小实现步骤及依赖方向。
5. 自动测试、实际运行和性能检查命令。
6. 文档与路线图更新项。
7. 停止条件和已知风险。

同一轮只允许一个里程碑。若发现更晚里程碑需求，只记录边界，不提前实现。

## 2026-07-11 · M1 游戏时钟和调度器

- 目标：建立唯一权威、按整游戏小时推进且与渲染帧率无关的游戏时间。
- 包含：配置化时间换算、1900 起始时间、暂停、1/2/4/8 倍速、单步、小时/日/周/月事件、稳定事件队列、时间控制 UI。
- 不包含：M2 实体模型、随机服务，以及任何人物、地图或存档逻辑。
- 设计：使用前推格里高利历；1900 年不是闰年。周事件从起点起每 168 小时触发。单个小时的事件顺序固定为小时、日、周、月、到期队列事件；同一到期小时的队列事件按插入顺序处理。
- 步骤：新增并验证时间配置 → 实现事件队列与权威时钟 → 以独立 Runner 接收帧增量 → 增加时间界面和菜单入口 → 扩展自动测试 → 实际启动和视觉检查。
- 验证：覆盖起始时间、暂停、合法速度、单步、日历边界、1900 非闰年、周期信号、队列排序/取消、不同帧分块结果一致、UI 场景加载。
- 性能：验证大块时间推进不会逐渲染帧运行业务逻辑；M1 仅记录测试和短时 8 倍速 UI 基线。
- 停止条件：M1 复选框全部通过后更新文档并停止，不进入 M2。
- 结果：已完成。项目累计 64/64 自动检查通过；时钟全年 8760 小时基线约 10.484 毫秒；headless 与 Compatibility 首帧检查正常。按计划停止于 M1。

## 2026-07-11 · M2 核心数据和随机种子

- 目标：建立不依赖场景节点的基础实体、稳定 ID、统一确定性随机服务和可报告错误的数据加载边界。
- 包含：国家、地区、控制单元、人口群体、人物基础、组织、关系、行动定义；`类型:标识` 稳定 ID；固定种子与 RNG 状态恢复；JSON 结构和交叉引用验证。
- 不包含：M3 的正式双国地图与拓扑玩法、M4 的人物随机生成和隐藏资质、M5 的行动结算、M8 的存档文件格式。
- 设计：模型使用强类型 `RefCounted`，只保存可序列化数据并提供 `to_dict()`。加载器先检查结构、类型和全局 ID 唯一性，再实例化并验证交叉引用；一次返回全部已发现错误。
- ID：静态及生成 ID 均采用小写 `命名空间:slug`；生成序号为八位十进制，计数器状态可导出和恢复。
- 随机：业务代码只能依赖 `DeterministicRandomService`；服务保存初始种子和 Godot RNG 状态，支持重设种子与状态恢复。默认种子来自独立 JSON 配置。
- 数据：M2 使用小型有效/无效 fixture 验证模型边界，不把 fixture 冒充 M3 正式世界数据。
- 验证：同种子同操作序列、RNG 状态恢复、ID 序列恢复、有效数据加载、重复 ID、错误类型和断裂引用。
- 性能：记录批量加载小型数据集和确定性随机序列的基础耗时；不代表 M3 规模性能。
- 停止条件：M2 三项复选框全部通过后更新文档并停止，不进入 M3。
- 结果：已完成。项目累计 104/104 自动检查通过；10 万次统一随机调用约 24.5 毫秒，10 实体 fixture 加载验证 100 次约 74.3 毫秒。按计划停止于 M2。

## 2026-07-12 · M3 二维地图和动态控制区

- 目标：用正式架空世界数据验证双层地图、控制变化、铁路和自动前线的完整基础闭环。
- 包含：2 个国家、8 个行政地区、80 个控制单元、4 座城市；拖动/缩放；法理边界、实际控制色、争夺纹理、铁路；地区/单元面板；压力导致易手；增量前线。
- 不包含：M4 人物选择与生成、完整军队编制、即时战术战斗、海空军或真实世界地理。
- 拓扑：使用 `10×8` 正交控制网格。东西各 5 列属于一个国家，每两行构成一个地区，因此每国 4 个地区、每地区 10 单元。四邻接预先写入正式 JSON，初始国境位于第 4/5 列之间。
- 城市与铁路：四座主要城市绑定到明确控制单元；铁路只连接相邻单元，包含一条东西干线和两条南北支线。
- 所有权：法理国家存于 `de_jure_country_id`，军事控制存于 `controller_country_id`，地区社会影响继续使用独立字典；任何服务不得合并为 `owner`。
- 控制阶段：阈值集中在 `data/balance/map_rules.json`，状态为稳定、减弱、争夺、敌方占领、巩固。压力只修改相关单元并在达到阈值时易手。
- 前线：启动时从邻接全量建立一次；之后控制者变化只重算该单元相邻边。边键由两个稳定 ID 排序生成，同一边只保存一次。
- 渲染：自定义 `Control` 只绘制 80 个单元，不为每个单元创建节点；实际控制决定填色，法理/地区边界用线条，争夺用程序斜纹，前线和铁路按边集合绘制。
- 交互：左键选择或拖动，滚轮缩放并以指针为中心保持位置；缩放和平移受配置边界约束。
- 验证：正式数据规模与引用、邻接对称、地图边缘、初始边境、易手与前线增删、孤立占领、包围、无邻居安全处理、铁路引用、缩放边界、地区摘要和 UI 场景。
- 性能：记录 80 单元数据加载、前线初始化、批量控制变化和 Compatibility 首帧。
- 停止条件：M3 三项复选框全部通过后更新文档并停止，不进入 M4。
- 结果：已完成。项目累计 158/158 自动检查通过；98 实体正式世界加载验证约 7.7 毫秒/次，800 次增量控制更新最终回归约 2.3 毫秒；Compatibility 首帧布局正常。按计划停止于 M3。

## 2026-07-12 · M4 随机人物和人物信息

- 目标：让玩家明确选择国家和随机模式，以固定种子获得可复现人物，并在正式/开发界面严格区分公开与隐藏信息。
- 包含：国家选择；标准、完整人口、分类三种随机模式；名字、地区、职业、年龄；六项隐藏资质、九项显性技能、性情权重和年龄显现性格；真实/已知倾向、当前状态；人物面板与开发数据切换；场景间玩家人物会话。
- 不包含：M5 行动进度与成功公式、M6 组织职位争取和 NPC AI、M8 存档。
- 配置：名字池、职业原型、模式权重、技能基础、倾向基础、年龄/资质范围和倾向事件全部集中在 `data/characters/character_generation.json`。
- 选择：国家必须由玩家明确选择，生成器拒绝空国家。标准模式只抽取有基本行动空间的职业；完整人口模式按人口群体选地区并按人口职业权重抽取，可产生贫困、失业等挑战人物；分类模式必须提供平民/军方/政界/工业界/知识界之一。
- 确定性：生成器注入统一 RNG 与稳定 ID 服务；同一世界数据、配置、种子、国家、模式和类别产生完全相同 `CharacterData`。生成不会调用全局随机函数。
- 信息：`CharacterData.to_public_dict()` 排除隐藏资质、性情权重和真实倾向。正式人物面板只显示玩家公开/已知数据；开发面板默认不可见，切换后才显示隐藏数值、真实倾向、生成种子和 RNG 状态。
- 性格：18 岁前无正式显现性格，18 至 24 岁显现 1 至 2 项，25 岁后显现 2 至 4 项；内部性情权重始终隐藏。
- 倾向：职业基础加小幅确定性变化；通过数据化事件增量修改并同步已知定性描述，不实现 M5 行动。
- 会话：静态 `GameSessionService` 仅保存当前玩家人物、选择和随机状态，保证人物预览、地图与人物面板切换时身份连续；M8 再持久化到文件。
- 验证：三模式、显式国家、类别过滤、同种子一致/异种子差异、年龄性格边界、资质修正范围、倾向事件、公开字典不泄密、开发面板默认隐藏及切换、地图入口和人物入口。
- 性能：记录批量人物生成与 UI 首帧；不生成 M6 背景人物群体。
- 停止条件：M4 三项路线图复选框全部通过后更新文档并停止，不进入 M5。

## 2026-07-12 · M5 行动和必然成功机制

- 目标：实现八类长期行动的配置、事件式进度、定性成功判断、中断和结果应用，并在战略地图提供可操作界面。
- 包含：学习、工作、建立关系、加入组织、争取职位、调查人物、推动地区政策、支持地区控制；工作量与效率；准备/资金/组织/关系/权限/状态/阻力输入；暂停、恢复、取消、中断、完成；六档成功把握；人物与控制区结果应用。
- 不包含：M6 的正式组织成员、职位结构、关系实体和 NPC AI；M7 的完整社会影响连接；M8 的行动存档。
- 数据：八类 `ActionDefinitionData` 加入正式世界；全局技能、效率、状态和把握分档集中在 `data/balance/action_rules.json`。
- 时间：行动实例只在查询、权威时间变化或依赖项变化时，从 `last_update_hour` 结算到当前整数小时；暂停和中断期间不积累工作量，不使用 `_process()`。
- 判定：有效值由技能、权限、组织/关系支持、准备、资金、隐藏资质温和修正、当前状态和阻力组成。达到普通阈值即成功，达到必然阈值为必然成功；本轮不加入不可控随机行动。
- 信息：正式 UI 只显示六档定性把握，不显示有效值或精确概率。测试可读取精确值验证阈值。
- 结果：学习、工作、声望和状态结果直接写回玩家人物；地区控制行动通过现有 `MapControlService` 施加局部压力；需要 M6/M7 实体的结果保存为已应用的领域效果描述，不提前创建组织或关系系统。
- 验证：八类定义、帧分块一致、暂停/恢复/取消、中断、依赖重算、完成、普通/必然阈值、结果只应用一次、地图控制结果和 UI 场景。
- 性能：批量查询行动进度和结算基线；每个活跃人物最多一个主要行动，不逐帧或逐小时扫描人物集合。
- 停止条件：M5 三项路线图复选框全部通过后更新文档并停止，不进入 M6。

## 2026-07-12 · M6 组织、职位、关系和简化 AI

- 目标：让四类组织、职位权限、按需关系和分层人物模拟可运行，并仅为活跃 NPC 执行有界的每日与长期 AI。
- 包含：政府、军队、企业、工会各两国一套；三级职位与权限；加入/离开、职位授予与降级；按人物对建立关系；120 名轻量背景人物；活跃层升级/降级和 20 人硬上限；活跃 AI 目标、候选行动、每日选择、每月长期评估和调试视图。
- 不包含：M7 的地区社会影响、人物退出与继承；M8 的完整开发工具和存档；完整选举、经济或军事编制。
- 数据：正式世界保存 8 个组织及职位结构；人物分层、关系默认值、AI 候选与权重集中在 `data/balance/society_rules.json`。
- 人物层级：背景人物使用独立轻量模型，不创建节点、不保存完整技能/隐藏资质/AI 状态；升级时以稳定激活种子补全 `CharacterData` 并保持 ID、姓名和经历字段，降级时恢复为同一稳定 ID 的轻量记录。玩家不可降级。
- 组织：运行时服务复制正式组织数据，维护成员和 `character → organization → position` 索引；技能不直接授予权限，权限只由当前职位定义产生。
- 关系：只为实际接触的人物对创建 `RelationshipData`，使用排序人物对作为索引；不存在的关系返回空，不建立全人物矩阵。
- AI：背景人物不进入 AI 状态表。活跃 NPC 每日只评估配置中有限候选；每月更新一个长期目标。相同状态下用稳定排序打破同分，不逐小时或逐帧决策。
- UI：地图社会面板显示组织结构、玩家成员/职位、人物层级计数、已知关系和活跃 NPC 的 AI 调试数据；升降级按钮是 M6 开发验证入口。
- 验证：四类组织、职位权限、加入/离开/职位槽位、关系唯一性与更新、120 背景/20 活跃上限、升降级身份连续、背景无 AI、每日/月度频率、同状态确定性、UI 和性能基线。
- 停止条件：M6 三项路线图复选框全部通过后更新文档并停止，不进入 M7。
- 结果：已完成。项目累计 387/387 自动检查通过；120 人社会初始化约 16 毫秒，30 日有界 AI 与 100 条关系约 6 毫秒；按计划停止于 M6。

## 2026-07-12 · M7 地区影响与人物继承

- 目标：把人物/组织作用连接到地区社会与军事状态，并让玩家人物退出后从真实关系网络选择继承者继续同一世界。
- 包含：政策行动改变地区社会影响；组织资源支持社会活动或军事控制；社会影响与军事控制分离验证；死亡、退休、长期监禁、严重失势、自愿退出；关系/组织候选；职位、财富、声望、盟友与敌对关系的部分继承；继承 UI。
- 不包含：M8 文件存档、安全写入和完整开发工具；家庭遗传、完整法律继承或自动剧情。
- 配置：社会影响幅度、组织资源消耗、五类退出原因、候选权重及各退出原因继承比例集中在 `data/balance/continuity_rules.json`。
- 地区：`RegionalInfluenceService` 只修改 `RegionData.social_influence`；军事支援只调用现有 `MapControlService`。政策行动不得改变控制者、控制强度、争夺度或前线。
- 组织：社会活动和军事支援消耗组织资源，并分别要求 `regional_policy`、`regional_control_support` 权限；人物技能不能绕开职位权限。
- 退出：退出人物从活跃/背景索引移动到独立已退出索引，保留稳定 ID 和完整退出记录；玩家当前行动取消，AI 状态移除。
- 候选：只从实际关系和共同组织成员中收集，同国、未退出、未拘押人物才可选；按熟悉、信任、好感、共同组织和声望数据化评分，稳定 ID 打破同分。
- 继承：候选必要时升级到活跃层并保持原 ID；财富转移并从前人物扣除，声望只部分传递；符合退出原因和最低评分时继承组织成员/职位；旧盟友和达到阈值的敌对关系按不同系数建立新关系，不复制全部人物数据。
- 连续性：`GameSessionService.transfer_player()` 只切换玩家引用并清空旧行动，不重建地图、组织、关系、AI 或随机会话。
- 验证：社会/军事分离、行动效果幂等、组织资源与权限、五类退出、候选来源/排序、继承比例、敌对保留、职位槽位、旧人物留档、新人物身份及世界对象连续、UI 和性能。
- 停止条件：M7 三项路线图复选框全部通过后更新文档并停止，不进入 M8。
- 结果：已完成。项目累计 471/471 自动检查通过；100 条关系候选约 1.0 毫秒，部分继承约 3.3 毫秒；按计划停止于 M7。
