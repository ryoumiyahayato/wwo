# P0-R1 模拟质量修复决定

日期：2026-07-13

本文件记录直接写入默认分支的第二批修复。它补充 `DECISIONS.md`，不表示本机验收已经通过。

## 人口抽样使用世界人口结构

- 决定：全人口人物先按国家总人口选择国家，再按人口群体人数选择地区和群体，最后用群体职业类别修正职业权重。
- 原因：固定两国轮换和全局职业表会让人物名册与世界人口数据脱节。
- 确定性：国家抽样与人物生成使用隔离的固定种子流，背景人物的 `activation_seed` 仍可独立恢复人物核心。
- 限制：职业倍率是 Demo 首版参数，需以 120 人分布和多种种子复验。

## 玩家准备采用显式财富投入

- 决定：玩家在行动开始前选择额外财富投入；基础费用和额外投入一次扣除，投入同时提高资金支持和准备度。
- 原因：原先准备与资金完全自动，玩家无法通过资源规划主动逼近必然成功。
- 存档：行动记录保存总费用、开始前财富和资金承诺；权威重算从已承诺总费用恢复额外投入。
- 限制：情报点仍作为自动准备因素，暂不单独消耗。

## NPC 使用同一长期行动模型

- 决定：活跃 NPC 保存 `ActionInstanceData` 字典，并按与玩家相同的费用、工作量、效率、阈值、结果和领域写回推进。
- 原因：每日瞬时免费效果让玩家与 NPC 处于两套规则，也无法解释行动耗时。
- 性能：NPC 只在每日边界批量推进，不逐小时扫描全部人物；背景人物不运行 AI。
- 关系自举：建立关系行动可从同国活跃或背景人物中选择新目标，不再要求已有关系才能维持关系。

## 组织经济保持资源循环

- 决定：组织按月获得由基础值、规模和影响力计算的收入，并受资源上限约束；社会和军事支援继续消耗资源。
- 原因：只有支出没有收入会让自主世界在若干月后永久停滞。
- 限制：当前不是完整预算、税收或市场模型，只保证 Demo 资源循环不会单向耗尽。

## 人物生命周期按年度结算

- 决定：每年一月边界统一增加存活人物年龄，老年人物降低健康；非玩家可退休或死亡并进入历史记录，组织领导空位随后补充。
- 玩家：达到退休或死亡条件时设置继承要求，不自动删除当前玩家。
- 原因：没有年龄和退出机制会让 1900 年世界长期模拟中的人物永久不变。
- 限制：尚未增加强制继承遮罩、生育或新人物补充，世界总人物身份只从存活层移入历史层。

## 地区行动必须有辖区

- 决定：政策只作用于人物国家的法理或实际控制地区；非政府组织的政策权限只覆盖其本地区；军事控制支援只允许政府或军队组织。
- 原因：职位权限不能自动等于全球权限，企业或工会也不应直接调动全国军事支援。
- 结算：UI 预检查和领域服务写回都执行辖区检查，避免只依赖前端限制。

## 控制压力读取物流和社会条件

- 决定：控制压力倍率读取相邻进攻方数量、进攻/防守铁路连接、地区社会影响和完全包围状态，并受最小/最大倍率限制。
- 原因：铁路、社会支持和包围此前只存在于数据或显示层，没有改变控制结果。
- 边界：当前使用地区级 `social_influence`，控制单元自身 `social_support` 暂未进入公式。

## 继承合并而非覆盖继承者

- 决定：继承关系在继承者原有关系基础上合并；原有组织身份保留；新增组织身份有配置上限；职位只允许升级，不得把继承者降到前人物较低职位。
- 原因：继承者是既存人物，不能被当作空白容器覆盖其社会历史。

## 页面不再决定世界是否运行

- 决定：人物页面也包含 `SimulationRunner`，并复用 `GameSessionService.world_clock`。
- 原因：人物资料页是表现层，不能因为切换页面就冻结权威世界和长期行动。
- 边界：暂停状态仍由权威时钟控制；页面只是提交现实时间增量。
