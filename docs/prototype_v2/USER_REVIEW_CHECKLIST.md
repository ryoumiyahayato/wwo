# V2.1.1 文化一致性与标签密度用户复核清单

V2 第一版用户视觉验收未通过。本清单用于重新检查 V2.1.1；截图和自动测试均不等于用户批准。

## 法国文化与制度语义

- [ ] 左上是否显示蓝白红圆形三色徽记，并且正式 UI 不出现内部国家代码？
- [ ] 国家详情是否显示“法兰西第三共和国”和 `République française`？
- [ ] 普通工人是否为“皮埃尔·勒费弗尔 / Pierre Lefèvre”？
- [ ] 地方官员是否为“阿尔贝·迪蒙 / Albert Dumont”？
- [ ] 职业、雇主、工会、工会职位、机构和机构职位是否分行显示？
- [ ] 我的组织和探索组织是否都使用里尔—北部省背景，并逐项解释接触来源？
- [ ] 北部省、里尔区、里尔市是否与“北部工业带”游戏宏观地区明确分开？
- [ ] 世界动态是否明确为静态原型条目，且没有旧架空地名？

## 三秒地图判断

- [ ] 三秒内能否认出北美洲、南美洲、欧洲、非洲、亚洲和澳大利亚？
- [ ] 大西洋、太平洋和印度洋是否有明确空间？
- [ ] 欧洲、非洲、亚洲和南北美洲的连续关系是否自然？
- [ ] 国家、国家内部地区和城市是否不再使用同一种标签层级？
- [ ] 法国近景是否仍遵守真实海岸线，而非方块拼图？

## 缩放与交通

- [ ] 世界远景、欧洲中景和法国近景是否确实显示不同对象？
- [ ] 标签是否按优先级裁剪，是否仍有难以辨认的重叠？
- [ ] 欧洲中景是否隐藏法国内部宏观地区并只保留 2–4 条主干铁路？
- [ ] 法国近景未选择对象时是否隐藏机构、组织和一般陆路？
- [ ] 选择里尔或北部工业带后是否只增加相应详细节点与标签？
- [ ] 铁路是否只连接陆地城市，并与一般陆路明显不同？
- [ ] 蓝色航运长虚线是否与铁路、国境和战线明显不同？
- [ ] 和平状态是否没有战线？
- [ ] 战争示例是否只像同一地图上的覆盖层？

## 四角界面与对象卡

- [ ] 打开任一主面板后，地图是否仍保留至少约 65% 可见范围？
- [ ] 396 px 侧栏是否仍然像大型后台管理面板？
- [ ] 人物中心打开时，左下是否不再重复显示人物摘要？
- [ ] 普通画面顶部是否没有常驻身份切换器？
- [ ] 地图模式是否压缩为单一入口？
- [ ] 单张对象卡是否只有一个主动作、有限次入口和“更多”？
- [ ] 页面是否主要依靠留白、字号、分隔和图标，而不是等宽矩形按钮？

## 身份与信息密度

- [ ] 普通工人页面是否明显以生活、工作合同、疲劳、雇主和工会为主？
- [ ] 地方官员机构页是否明确显示部门层级、上级、协作单位、管辖、预算、议程和程序？
- [ ] 同一机构在工人公开视角和官员内部视角下是否结构不同？
- [ ] 我的组织是否显示徽记、职位、部门、项目、上级和权限摘要？
- [ ] 探索组织是否逐项解释工作、地区、关系、新闻或公开记录来源？
- [ ] 人物、组织、机构、地区和行动是否都遵守扫视 / 悬停 / 点击详情 / 隐藏 QA 四层密度？

## 世界动态与输入

- [ ] 右下默认是否只显示一条重要通知、未读数和展开入口？
- [ ] 历史展开后是否能区分通知、事件和新闻，并聚合同类消息？
- [ ] 左键对象选择、地图拖动、滚轮缩放、面板互斥和 Esc 是否符合直觉？
- [ ] 1280×720 是否没有裁切、点击穿透或严重文字重叠？

用户只有在实际运行并明确同意后，才算批准 V2.1.1 文化、制度语义与标签密度方向。在此之前不得接入任何正式业务系统。
