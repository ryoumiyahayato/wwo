# V2.1 静态原型交互规范

## 地图优先与四角入口

- 左上：当前国家或所属机构入口。
- 右上：日期为主，显示明确的暂停/运行状态与当前速度；点击后才展开自动暂停和系统工具。
- 左下：当前人物的紧凑角标；人物中心打开时该角标不再绘制。
- 右下：最近一条重要通知、未读数和展开入口；不是常驻历史文本框。
- 下方中央：单个当前地图模式入口；点击才展开四种覆盖层。

国家、人物、时间和动态主面板互斥。国家、人物与动态侧栏宽 396 px，时间面板宽 292 px；1280×720 下均不超过 420 px，横向保留超过 65% 的地图可见范围。

## 对象驱动交互

主要操作从地图国家、地区、城市、港口、机构、组织和关系人物进入。不存在独立“大行动中心”。对象卡第一层最多提供：

- 一个主要动作；
- 一个文字次要动作；
- 一个“更多”入口。

人物第一层是“联系 / 请求帮助 / 更多”；引荐、调查和共同关系不再平铺为等权按钮。第三层详情才显示完整关系结构与来源。

## 四层信息密度

1. 扫视：对象名称、层级、一个关键状态和有限动作。
2. 悬停：来源、权限、线型或限制的一句说明。
3. 点击详情：完整已知结构、因果来源与上下文动作。
4. 隐藏 QA：内部 ID、原始字段与公式不进入普通画面。

这一规则同时适用于人物、组织、机构、地区、动态和行动。

## 身份结构

普通工人人物中心以生活、收支、工作合同、疲劳、关系、雇主、工会和个人计划为主。国家入口只展示公开政策、公开新闻和可接触组织。

地方官员除个人信息外，机构中心使用组织层级图，明确所属部门、上级、协作单位、管辖地区、预算来源、当前议程、可执行程序，以及“已知但无权处理”的上级事项。两者不是同一页面增删按钮。

身份切换只属于 `--prototype-review` 评审工具；本地评审也可用 `F9` 临时显隐。普通运行和普通截图中不显示。

## 地图输入与缩放语义

- 地图空白处左键拖动平移；滚轮围绕指针缩放。
- UI 命中时不平移、不缩放；拖动超过阈值后不触发对象选择。
- 远景显示国家、国家名、少量主要城市和海洋。
- 中景显示法国宏观地区、主要城市、港口、主干铁路与航运。
- 近景显示小城市、地方机构、组织节点、支路和一般陆路。
- 标签按优先级与碰撞矩形裁剪，不随缩放无条件放大同一批文字。

法理、市场、人口和战争状态共用 `natural_earth_110m_v2_1` 底图，只更换覆盖层。和平状态不绘制战线；战争示例仅增加静态控制覆盖、战线与战区标签。

## 交通语义

- 铁路：深色双线、金色芯线和横向枕木，只连接陆地城市。
- 一般陆路：细短虚线。
- 航运：蓝色长虚线，连接港口并可跨海。
- 国境：浅色细实线。
- 战线：红色粗线与半透明控制覆盖。

## 关闭层级与动画

`Esc` 依次关闭模式菜单、人物/行动详情、地图对象卡和主要中心。无上层界面时不会退出原型。主要中心使用约 200 ms 四次 ease-out 打开、约 160 ms 关闭，无弹跳和点击穿透。
