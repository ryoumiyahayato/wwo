# 真实世界地图数据与 Godot 2D 渲染

状态：设计冻结 v1。正式方向是 1900 年真实世界地图；当前 10×8 架空网格只保留为技术历史，不继续扩充或美化。

## 1. 首版世界范围

- 完整世界海岸线与主要内陆水体。
- 1900 年开局时点的国家、殖民地与争议地区。
- 约 150～250 个宏观地区；首版内容目标为 180～220，允许数据验证后在上限内调整。
- 约 300～500 个主要城市/地方节点，按缩放和玩家附近显示。
- 约 100～180 个主要港口。
- 简化但连通的 1900 年主要铁路网，目标 1500～3000 条预处理线段，不追求每条支线。
- 玩家所在国家及邻近地区可加载更细的地方节点；其他世界保持宏观模拟。

国家、地区、地点、铁路、港口和水域都使用稳定 ID。真实名称与本地化名称分开保存。

## 2. 历史时点与数据来源政策

地图权威开局时点为 1900-01-01。边界必须区分：

- 法理主张。
- 国际承认或行政归属。
- 实际控制。
- 殖民/附属关系。
- 争议状态。

所有外部地理与历史数据在进入仓库前必须记录来源、版本、许可、修改方法和核验日期。只接受明确允许离线再分发与修改的来源；来源不明素材禁止进入项目。

每个数据包提供 `provenance` 清单和转换脚本说明。最终来源选择、许可复核与历史边界校订属于进入真实地图阶段前的实现门；不得让后续代理自行下载无来源地图。

## 3. 静态数据分层

```text
WorldMapDataset
  metadata
  water_bodies[]
  countries[]
  territories[]
  macro_regions[]
  localities[]
  infrastructure_nodes[]
  infrastructure_links[]
  spatial_index
  geometry_lods
  provenance[]
```

### 国家与领地

保存国家 ID、正式名、政体引用、首都、领土/附属关系和历史有效期。

### 宏观地区

是市场、人口、地方机构、迁移和部分政策的主要聚合单位。边界优先依据历史行政、经济联系、人口中心和地理屏障，不只按面积平均切分。

### 地方节点

城市与地方尺度使用点/范围节点，保存住区、工业、行政、港口、铁路枢纽和重要机构标签；首版不绘制真实街道。

### 基础设施

铁路、港口、道路和河运链接保存几何、容量、速度、建设/关闭时间、所有者、状态和连接节点。地图画线只是该权威链接的视图。

## 4. 动态状态分离

静态地理不进入存档。运行时只保存：

- 当前实际控制与占领。
- 边境通行和战争状态。
- 基础设施建设、损坏与容量变化。
- 地区市场、人口、组织影响和技术采用摘要。
- 城市/地点的运行状态。

法理、控制、市场范围和组织影响是不同图层，禁止合并为单一 `owner`。

## 5. 投影与坐标

世界尺度采用预处理的 Robinson 投影坐标，兼顾完整世界可读性和海陆形状。运行时不重复投影经纬度；所有多边形和线段在导入阶段转换为归一化世界坐标。

路线距离、运输成本和军事距离使用独立的地理/网络长度字段，不从投影后的屏幕像素反推。

国家与地方尺度继续使用同一世界坐标并放大；需要更细地方图时加载该区域的独立 LOD 几何与抽象功能区。

## 6. 几何处理流水线

```text
许可明确的源数据
→ 坐标与历史时点规范化
→ 拓扑修复、简化和多级 LOD
→ Robinson 投影
→ 生成三角网格、边界线、空间桶与稳定 ID 索引
→ 结构/引用/几何验证
→ Godot 只读运行时数据包
```

预处理可使用仓库内受控工具，但正式运行时不依赖第三方插件或在线服务。生成物必须可由源清单与脚本重复构建。

## 7. 语义缩放

### 世界尺度

显示国家、首都、大型市场、外交、战争和技术扩散。隐藏地方组织和普通人物。

### 国家/地区尺度

显示宏观地区、主要城市、铁路、港口、企业中心、人口和地方机构。

### 城市/地方尺度

显示抽象住区、工业区、政府区、港口、车站、主要组织和当前人物可达对象，不进入街道模拟。

LOD 切换使用屏幕像素密度与滞回区间，避免缩放边界闪烁。标签有稳定优先级和碰撞布局，不因每帧随机排序跳动。

## 8. 地图模式

- 法理归属。
- 实际控制。
- 市场与运输。
- 人口与迁移。
- 组织影响。
- 基础设施。
- 技术形成/扩散/采用。
- 战争、动员与补给。

地图模式只改变视图和可选信息，不改变玩家权限。所有颜色配套纹理、图标或边界差异，不能只依赖红绿辨识。

## 9. Godot 2D 渲染方案

使用 Compatibility 渲染器和少量自定义 `Control`/CanvasItem 绘制层，不为每个地区、人口或背景人物创建可见节点。

建议层级：

```text
MapRoot
  WaterLayer              静态批次
  TerritoryFillLayer      缓存三角网格，模式变化时更新材质参数
  BorderLayer             国境/地区/实际控制线批次
  InfrastructureLayer     铁路、港口与道路的 LOD 批次
  DynamicWarLayer         战区、前线、补给和控制变化
  LabelLayer              池化标签
  IconLayer               池化城市、组织、人物和军队图标
  SelectionLayer          悬停、选择、路线预览
```

多边形在预处理阶段三角化并按 LOD、图层与颜色索引批次缓存。平移和缩放主要修改画布变换；只有模式、LOD 或动态状态变化才重建相关批次。

## 10. 命中与标签

命中测试使用预处理的空间网格桶：先按世界坐标取得候选地区，再做点在多边形内测试。城市、组织和军队图标使用屏幕空间索引，并按当前层级和优先级解决重叠。

标签优先级：玩家地点/选择对象 → 国家/首都 → 主要城市 → 宏观地区 → 普通地方。隐藏标签仍可通过搜索或对象列表访问已知信息。

## 11. 前线与动态边界

和平状态不生成战争前线。显式战争创建战区和前线图层；前线由相邻实际控制区域或作战区段增量维护，不每帧扫描全世界边界。

控制变化只使相关边段、填色和命中摘要失效。国境、法理边界与前线使用不同视觉语义。

## 12. 多精度模拟

### 活跃命名人物

玩家周边、主要组织和关键职位人物拥有完整状态、知识、时间与计划。

### 背景命名人物

保存身份、职业、组织、关系摘要、住户摘要和低频目标；进入玩家认知或关键职位时激活。

### 人口群体

按地区/职业/阶层聚合人口、收入、需求、就业、迁移、健康和支持。普通人口变化不创建逐人节点。

### 空间活动区

玩家周边与重大危机地区提高更新精度；远方地区按月/季度聚合。精度改变只改变结算分辨率，不改变总量守恒和稳定 ID。

## 13. 缓存与更新

- 海岸线与静态地形：加载后缓存。
- 模式填色：模式或对应数据版本变化时更新。
- 标签：缩放、视口或高优先对象变化时重新布局，受控频率。
- 前线：战争与控制相关边变化时增量更新。
- 市场/人口热力图：月度摘要变化时更新纹理或颜色缓冲。
- UI 不在每帧查询全部实体。

## 14. 性能预算

设计目标环境仍为 Windows 10/11 x86-64、Godot 4.6.3 Compatibility、1280×720 最小逻辑分辨率。

| 项目 | 设计预算 |
| --- | --- |
| 平移/缩放交互 | 目标 60 FPS；单帧总预算 16.7 ms |
| 地图 CPU 更新 | 常规交互低于 4 ms/帧 |
| 模式或 LOD 重建 | 分块执行；单帧低于 8 ms，避免超过 50 ms 停顿 |
| 悬停命中 | 低于 1 ms 常规、2 ms 最坏目标 |
| 世界静态地图内存 | 首版目标低于 128 MiB |
| 可见图标节点 | 池化，常规低于 500 |
| 宏观地区 | 150～250 |
| 简化铁路线段 | 1500～3000 |

这些是实施验收预算，不是已测结果。真实数据进入后必须在指定引擎上重新建立基准。

## 15. 扩展行政区

未来增加省州或更细地区时：

- 保留父级宏观地区稳定 ID。
- 子区具有历史有效期与父引用。
- 市场、人口和存档可继续以宏观地区聚合。
- 新细节作为 LOD 和局部模拟扩展，不要求一次重写全球内容。
- 旧存档通过父级映射迁移动态状态，不能按数组位置映射。

## 16. 数据验证

导入检查：ID 唯一、几何闭合、无自交或明确修复、父子引用、国家/地区覆盖、历史有效期、铁路端点、港口水域连接、LOD 对应、许可清单完整。

运行检查：命中稳定、标签不抖动、模式不修改权威状态、和平无前线、增量更新范围正确、不同精度总量守恒。

## 17. 首版与保留接口

首版必须：完整世界海岸线、1900 国家、150～250 宏观地区、主要城市/铁路/港口、三尺度、八类模式、空间命中、多精度模拟和上述预算。

只保留接口：街道级地图、全部县级行政区、逐列车运行、逐建筑城市、3D 地形和在线地图瓦片。正式游戏保持完全离线。
