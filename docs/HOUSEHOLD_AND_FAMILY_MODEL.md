# 住户、住房与家庭模型

状态：设计冻结 v1。住户是人物生活经济和社会连续性的基础实体；本轮不修改正式存档，也不实现婚育系统。

## 1. 设计目的

人物不能被视为孤立的现金账户。住户统一承载住房、共同收入支出、抚养与照料、家庭债务、迁移和成员互助，使农民、工人、贫困者与家庭成员拥有完整玩法。

每个存活人物必须恰好属于一个当前住户。独居人物属于单人住户；无住处者仍属于“无固定住所”住户，而不是没有住户。

## 3. 财务边界

人物保留个人现金、个人资产和个人债务。住户拥有共同账户、共享资产和家庭债务。每笔收入按 `finance_agreement` 分配为个人留存和住户贡献，不得把所有成员财富简单合并。

住户资金流：

```text
成员贡献 + 资产收益 + 救济/转移
- 食品与燃料消费 - 房租/维护 - 照料 - 税费 - 家庭债务偿还
= 住户现金变化
```

不足时按以下顺序处理：

1. 降低非必要消费。
2. 使用住户现金缓冲。
3. 按协议请求成员额外贡献。
4. 使用合法信用额度或亲友援助。
5. 形成欠款、降低生活水平或触发住房风险事件。

不得凭空补齐缺口。任何救济、借款或援助必须有付款主体和对应资金流记录。

## 4. 生活需求

住户根据成员年龄、健康、工作强度、地区气候和生活水平生成月度消费篮子。首版商品聚合为食品、住房、燃料、衣物与日用品，不要求玩家逐件购买。

生活水平是住户选择与可负担能力共同形成的目标：

- 极度节俭
- 节俭
- 普通
- 舒适
- 奢华

实际满足度与目标分开保存。选择舒适生活但无法购买消费篮子不会产生虚假健康收益，反而形成缺口、债务和压力。

## 5. 家庭与照料义务

照料包括儿童、病患、老人和临时危机。它消耗住户成员的每周时间预算，并可通过成员分工、付费服务、邻里互助或公共服务缓解。

照料不足可能导致：

- 被照料者健康或教育受损。
- 照料者压力增加。
- 家庭关系冲突。
- 工作缺勤或计划暂停。

富裕人物可以购买服务释放时间，但服务费用必须进入提供者或市场主体，不能只作为抽象时间兑换。

## 6. 住房作用

住房同时影响：

- 每月支出和资产维护。
- 健康风险与疾病恢复。
- 疲劳恢复和基础休息质量。
- 通勤时间。
- 地方可接触人物与组织。
- 地方公众评价的有限修正。

失去住房不是 Game Over。可能进入寄住、廉价租房、雇主宿舍、机构安置、非正式住处或无固定住所状态，并打开至少两条恢复路径。

## 7. 迁移与住户变化

迁移是个人计划或反应事件，不是瞬时地区 ID 修改。迁移需要目的地、交通、费用、旅行时间、住房安排和成员同意；跨地区可能中断就业与弱关系维护。

住户可因以下情况拆分或合并：

- 结伴居住、婚姻或亲属同住。
- 子女或成员独立。
- 工作迁移。
- 战争撤离或难民安置。
- 关系冲突、驱逐或住房丧失。
- 人物死亡或长期失踪。

拆分时按财产权、协议、债务责任和制度裁决转移资产，不采用平均复制。

## 8. 与社会继承的关系

住户成员是继承候选来源之一，但不是自动继承者。候选仍需满足存活、可接触、信任或正式接班安排等条件。

社会继承分别处理：

- 玩家控制权：转移到合法社会继承者。
- 财产：按所有权、债务、遗嘱或制度规则转移。
- 住户身份：继承者可能留在原住户、加入另一住户或建立新住户。
- 关系与责任：照料、债务担保和家庭义务可继续存在。

四者不得被一个“继承比例”同时替代。

## 9. 更新周期

| 周期 | 结算内容 |
| --- | --- |
| 即时 | 搬入搬出、失去住房、重大支出、成员退出 |
| 每日 | 基础消费满足摘要、照料缺口、住房健康风险 |
| 每周 | 时间分工、照料义务、家庭关系与通勤负担 |
| 每月 | 收入贡献、消费篮子、房租、维护、债务和生活水平 |
| 每年 | 长期住房、抚养阶段和社会继承候选重验 |

每日不逐件扣除食品；市场和住户在月度结算形成权威消费，日结算只处理已经存在的短期缺口与特殊事件。

## 10. AI 使用

AI 的生存与家庭安全目标读取：住户现金缓冲、消费缺口、住房风险、被抚养者、照料缺口和成员收入稳定性。AI 先维持住户与合同义务，再评估高风险计划；不会因随机候选而长期忽略饥饿、无住处或病患。

住户级决定由可决策成员按关系、贡献、权利和紧迫性协调。首版允许一个确定性代表决策者，必须保留多成员同意接口。

## 11. UI

人物中心第一层只显示：住户规模、住所状态、生活水平、月度结余和最紧迫家庭负担。点击“生活”进入住户详情，显示成员卡、住房、共同财务、照料和最近原因记录。

不显示原始消费权重。现金不足、住房风险或照料缺口以一行状态和两条可行入口呈现。

## 12. 首版与保留接口

首版必须：单人/多人住户、住所、共同账户、月度消费、成员贡献、照料小时、生活水平、住房风险、迁移和继承候选连接。

只保留接口：婚姻法、复杂生育、遗传、离婚财产诉讼、完整教育抚养、房屋细分产权与多套住宅组合。首版不显示这些未实现功能。
