# 最终 UI 信息架构与 1280×720 线框图

状态：设计冻结 v1。本文只定义 UI 空壳和信息流，不接入当前正式会话。所有线框均为 1280×720 逻辑像素的低保真布局。

## 1. 空间原则

地图始终占据整个窗口背景，不再缩成中央小框。四角只放高频、可扫视组件：

- 左上：国家与主要机构。
- 右上：时间与系统工具。
- 左下：当前人物中心。
- 右下：世界动态与通知。

点击角落组件或地图对象后，在对应方向展开卡片/工作区，地图仍可见并保留空间上下文。同一侧只保留一个主要工作区；事件决策可以在其上形成最高层卡片。

## 2. 基准坐标与尺寸

| 组件 | 基准矩形 `(x,y,w,h)` | 展开方向 |
| --- | --- | --- |
| 国家与机构 | `(16,16,304,76)` | 向下/向右 |
| 时间与工具 | `(1000,16,264,88)` | 向下 |
| 人物中心 | `(16,604,360,100)` | 向上/向右，主面板宽 480 |
| 世界动态 | `(880,500,384,204)` | 向上/向左，历史宽 440 |
| 地图对象卡 | 选择点附近，默认 `360×420` | 避开四角组件 |

安全边距 16，组件间距 8。缩放到更大分辨率时地图扩展，角落保持锚定；文本和卡片按 UI 缩放设置变化。1280×720 是首版最小逻辑分辨率。

## 3. 输入与导航

- 左键：主要操作；选择人物、地点、地区、组织、国家、军队或 UI 卡片。
- 右键：地图返回/取消路线预览，不能承载唯一功能。
- 滚轮：地图缩放；卡片悬停时只滚动卡片内容。
- 拖动：平移地图。
- `Space`：输入控件未占用时暂停/继续。
- `Esc`：按层级关闭事件卡、详情、工作区、对象卡；连续按不会直接退出游戏。
- 点击已打开的角落入口：收起工作区。

地图选择只改变观察对象，不隐式改变个人计划或行动目标。

## 4. 上下文行动

不设置全局八按钮行动页。动作出现在对象卡和身份工作区：

- 人物中心：学习、求职、休整、迁移、医疗、投资。
- 人物卡：联系、拜访、帮助、引荐、谈判、调查、合作。
- 组织卡：申请、退出、查看职位、争取职位、参与项目。
- 机构/职位卡：议程、预算、任命、执行、动员、军事命令。

无关行为隐藏；已知但条件不足的行为显示锁和一行原因。服务层返回可用性，UI 不复制权限公式。

## 5. 四层信息密度

1. 扫视：图标、名称、短状态、趋势。
2. 悬停：当前值、阈值、主要原因、信息来源。
3. 点击：完整已知来源、历史、对象与上下文行为。
4. QA：内部 ID、原始字段、公式、权重、随机状态，仅 Debug/`--qa`。

正式 Release 完全移除可见开发者入口。

## 17. 卡片状态与动效

- 角落工作区进入/退出建议 180～250ms ease-out，不作为权威状态边界。
- 通知从右下滑入，普通消息自动淡出，重要消息保留更久。
- 动效期间输入目标保持稳定；禁用或遮挡控件不得响应。
- 减少动态设置下改为瞬时切换或短淡入。

## 18. QA 与正式 Release

Debug 构建、命令行 `--qa` 或明确的本地开发配置可以启用隐藏控制台；启用后使用反引号打开。建议首批命令冻结为：`inspect_character`、`inspect_region`、`inspect_organization`、`set_age`、`set_health`、`set_cash`、`set_position`、`trigger_event`、`start_war`、`end_war`、`advance_time`、`validate_save`。

正式 Release 不创建可见开发工具按钮，不显示内部 ID、随机种子、原始权重、调试状态或直接改写世界的控件。

## 19. UI 空壳原型验收

下一阶段只用静态假数据验证：四角锚定、全屏地图、11 类工作区/卡片、悬停、左键选择、Esc 层级关闭、1280×720 无裁切、缩放适配和减少动态。不得连接正式社会模拟或修改正式存档。

## 20. 首版与保留接口

首版必须：全屏地图、四角组件、本文 11 类线框、人物/组织/地点对象卡、上下文行为、四层信息密度、聚合动态、键盘关闭和 1280×720 适配。

只保留接口：玩家自定义布局、多窗口、复杂报表编辑器、手柄径向菜单、触屏专用布局和完整无障碍旁白。保留接口不产生空入口。
