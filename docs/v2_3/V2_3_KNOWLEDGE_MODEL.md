# V2.3 人物有限认知

## 真实状态与人物认知

真实状态由空间、旅行、消息、人物、组织、关系和约见服务保存。人物认知是
`KnowledgeService` 中按人物隔离的事实记录，不是世界状态的副本。普通游戏查询必须经过
认知过滤；只有明确标记的评审/开发者真相视角可以只读真实状态。

每条知识保存事实类型、主题、值、来源类型、来源 ID、获知时间、可信度、过期时间、状态、
关联消息 ID 和稳定知识 ID。人物可见状态至少包括：

- `rumor`
- `reported`
- `confirmed`
- `expired`
- `contradicted`

未知事实没有记录，不以空值或隐藏标签泄漏。

## 来源与可信度

| 来源 | 可信度 |
|---|---:|
| 直接观察 | 1000 |
| 组织通知 | 950 |
| 面对面 | 900 |
| 私人信件 | 800 |
| 公开公告 | 750 |
| 传闻 | 450 |

可信度按 0–1000 限幅。新证据使用稳定来源优先级和时间比较更新现有事实；矛盾事实保留来源
与因果，不把低可信消息静默提升为真相。

## 时效

- 人物位置：6 小时
- 人物日程：24 小时
- 组织成员身份：720 小时
- 身份：不过期
- 路线知识：720 小时
- 公开公告：168 小时

到期事实标记为 `expired` 并保留历史，不从世界真实状态重新补全。每人物知识历史最多 256
条。送达未读消息不产生事实，阅读、直接观察或实际面对面互动才会写入。

## 已知地点与人物

初始已知地点来自人物住所、工作地、明确默认知识和可解释邻接；初始已知人物来自配置关系，
不从全世界人物表推导。皮埃尔初始只认识让娜；阿尔贝初始只认识吕西安，不认识让娜或儒勒。
介绍消息送达但未读时，陌生人仍不可见；阅读正式介绍后才加入已知人物和联系人。

普通地图不显示未知地点名称，联系人页不列陌生人，消息目标也不能直接选陌生人。评审模式
允许切换皮埃尔/阿尔贝来检查不同认知，但切换不重置任何模拟状态。
