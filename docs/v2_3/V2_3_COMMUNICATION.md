# V2.3 延迟通信

## 渠道

| 渠道 | 写作 | 本地延迟 | 区域延迟 | 费用 | 约束 |
|---|---:|---:|---:|---:|---|
| `face_to_face` | 1 小时 | 0 | 0 | 0 | 双方同地 |
| `local_letter` | 1 小时 | 12 小时 | 24 小时 | 5 生丁 | 可投递地址 |
| `organization_notice` | 0 | 6 小时 | 6 小时 | 0 | 组织成员 |
| `public_notice` | 0 | 0 | 0 | 0 | 实际观察或订阅 |

渠道数值来自 `communication_channels.json`，均标记为 Demo 平衡值。

## 消息生命周期

私人信件依次经历 `drafted`、`in_transit`、`delivered_unread`、`read`，回复是具有
`reply_to_message_id` 的新消息。发送、送达、阅读和回复分别使用稳定幂等键；重复调用不会
重复扣邮资、重复投递、重复写入知识或重复改变关系。

“送达”和“阅读”严格分离：

- 到达时间由发送小时、同城/区域和渠道配置确定；
- 到达只改变收件箱状态和未读数；
- 阅读才把消息携带的事实交给 `KnowledgeService`；
- 私人回复只有在收件人实际阅读并创建回复后，才产生较弱关系变化；
- 消息内容、状态、到达/阅读时间和回复链全部进入 V2.3 存档。

## 组织与公开信息

组织通知只发给发送时符合成员条件的接收者，并经过固定六小时延迟。公开公告作为世界真实
信息存在，但人物必须到公共广场实际观察，或满足配置的订阅条件，才能阅读并取得知识。切换
观察人物不会共享收件箱、未读状态或知识。

## 性能与边界

送达队列按到达小时和稳定消息 ID 排序，只在权威小时边界处理到期项。消息历史上限为 256，
幂等键总上限为 1024。通信状态变化不会重建国家几何、行政区或地点投影目录。本轮不实现完整
邮政经济、电话网络、媒体产业、报纸编辑或复杂调查系统。
