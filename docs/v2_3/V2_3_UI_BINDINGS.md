# V2.3 UI 与地图绑定

## 组合方式

V2.3 主场景继承现有 V2.2 主场景与四角界面，通过 `V23LifeLoopUiBinding` 把命令转发到
`V23LifeLoopSimulation`，并读取过滤后的只读状态。UI 不计算路线、扣费、改变位置、写知识
或结算关系。

新增界面包括：

- 当前位置、途中状态和预计到达
- 已知目的地、最快/最省路线预览与确认
- 旅行段日程和迟到提示
- 动态联系人、联系渠道、介绍请求和约见
- 消息中心、收发状态、未读数、阅读和回复
- 个人认知日志、来源、可信度、时效和过期状态
- 评审人物切换与明确标记的开发者真相视角
- V2.3 保存、加载和 V2.2 迁移入口

阻断面板沿用 V2.2 暂停/恢复语义，Esc 逐层关闭。时间编辑区保持紧凑；1280×720 不依赖
横向滚动显示核心按钮。

## 认知过滤

绑定始终携带 `selected_person_id`：

- 地点选择只返回该人物已知且可达的地点；
- 联系人和消息目标只返回已知人物与可用渠道；
- 知识日志只返回该人物记录；
- 普通地图隐藏未知地点名称与无知识来源的他人位置；
- 评审模式允许切换皮埃尔/阿尔贝，但切换只改变观察主体；
- 真相视角带醒目标记，且不反向写入人物知识。

## 世界地图

V2.3 复用唯一 `WorldMapCanvas`。Lille 本地点位、已知边、路线预览、当前路线、人物位置和
标签作为现有交通/节点/选择/标签批绘层的增量内容。只在 48 倍以上查询本地点，72 倍以上
绘制标签，最大缩放仍为 96。

地图缓存配置版本对应的投影目录和均匀网格索引。旅行位置变化只更新本地覆盖层；消息、知识
和关系变化不会触发国家几何、行政区或投影重建。性能调试快照暴露目录重建、投影调用、空间
查询候选和分层重绘计数，供自动测试和真实窗口复验使用。
