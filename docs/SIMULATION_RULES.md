# 模拟规则

本文件集中记录临时公式、阈值和结算频率。权威时间规则已在 M1 实现并验证；领域结算仍是后续里程碑的设计输入。

## 权威时间

- 状态：M1 已验证；现实秒换算仍待完整游戏平衡。
- 开始时间：1900-01-01 00:00。
- 最小权威单位：1 游戏小时。
- 暂定换算：1 现实秒 = 1 游戏小时，存储于 `data/balance/simulation_clock.json`。
- 速度：暂停、1、2、4、8 倍。
- 日历：前推格里高利历，1900 年不是闰年。
- 周边界：从起点起每 168 小时，在午夜触发。
- 帧率规则：现实时间可按任意帧分块输入；只提交完整小时并保留不足一小时的余量。

## 调度顺序

每个游戏小时先更新权威日期，再按以下固定顺序结算：

1. 小时事件：长期行动等逐小时权威状态先结算。
2. 若跨日，发出日事件。
3. 若跨月，发出月事件。
4. 处理该小时到期的队列事件。
5. 若达到 168 小时周边界，最后发出周事件并执行自动存档。

这样批量推进和逐小时推进得到相同的行动、AI、地图与存档状态；周自动档不会早于同一小时的月度世界结算。队列事件 ID 唯一；相同到期小时按插入顺序执行。载荷在入队和发出时深拷贝，避免调用方随后修改影响确定结果。

## 确定性随机数

- 默认开发种子：`19000101`，位于 `data/balance/random_service.json`。
- 所有业务随机调用统一通过 `DeterministicRandomService`。
- 相同种子和相同调用顺序必须产生相同整数、浮点、选取和洗牌结果。
- 服务可读取及恢复 Godot RNG 状态；只保存初始种子不足以恢复已进行的序列。
- 稳定 ID 不消耗随机数，避免实体创建顺序无意改变随机序列。

状态：M2 已验证。玩家设置种子和开发者界面属于后续里程碑。

## 地区控制与前线

阈值集中在 `data/balance/map_rules.json`，M3 当前值：

- 控制减弱：法理方控制强度低于 `0.55` 且争夺度低于 `0.45`。
- 争夺中：争夺度达到 `0.45`。
- 敌方占领：军事控制者不同于法理国家、争夺度低于 `0.45`、控制强度低于 `0.55`。
- 控制巩固：军事控制者不同于法理国家、争夺度低于 `0.45`、控制强度不低于 `0.55`。
- 稳定控制：军事与法理一致，且未进入减弱或争夺。

一次强度为 `1.0` 的敌方压力暂定：控制强度 `-0.12`、争夺度 `+0.18`、敌方压力 `+0.15`。当控制强度不高于 `0.15` 且争夺度不低于 `0.65` 时易手，新控制强度为 `0.30`，旧进攻压力清零，争夺度降入巩固区间。控制方继续投入会增强控制并降低争夺。组织 AI 会优先巩固仍处于前线的薄弱己方单元，再向新的敌方前线投入。

相邻单元军事控制者不同且均非空时形成前线。前线与法理归属、社会影响分别计算；法理变化或社会影响变化本身不移动前线。

状态：M3 已通过易手、增量前线、孤立、包围和前线消失测试；数值仍待后续玩法平衡。

## 资质修正

- 暂定范围：`0.80` 至 `1.20`。
- 状态：临时、未实现、待平衡。
- 资质不构成绝对技能上限。

## 行动有效值

```text
技能 + 职位权限 + 组织支持 + 关系支持 + 准备 + 资源投入
+ 隐藏资质小幅修正 + 当前状态修正 - 目标阻力
```

M5 已实现以下首版结构：

```text
主要技能 × 0.65 + 次要技能平均 × 0.15
+ 职位权限加值
+ 组织支持 × 行动权重 + 关系支持 × 行动权重
+ 准备 × 行动权重 + 资金 × 行动权重
+ (对应隐藏资质 - 50) × 温和权重
+ 当前状态值 × 行动权重
- 基础目标阻力 - 额外目标阻力
```

全局权重、状态公式、进度倍率和定性把握分档位于 `data/balance/action_rules.json`；各行动工作量、技能、局部权重、阈值和结果位于正式世界行动定义。有效值达到普通阈值成功，达到必然阈值必然成功；正式 UI 只显示六档定性文本。

行动按 `last_update_hour` 到当前权威整数小时的差值结算。依赖变化时先按旧效率结算旧区间，再保存新输入并重算；暂停、取消或中断后不推进。状态：M5 已验证，数值仍需后续玩法平衡。

## M6 活跃 AI

- 背景人物：不创建 AI 状态，不参与每日或月度候选评分。
- 活跃 NPC：每日最多评估配置中的 5 个候选；玩家不由 AI 控制。
- 长期目标：在实际月边界更新，不用固定 30 天替代日历月。
- 评分：候选基础权重 + 相关技能温和权重 + 当前长期目标加成 - 疲劳/压力惩罚；恢复候选随疲劳和压力增加。
- 同分：按行动稳定 ID 排序，不消耗随机数。
- 频率与权重位于 `data/balance/society_rules.json`。状态：M6 已验证，候选和平衡值仍为首版。

## M7 地区影响与继承

- 政策行动：默认提高执行人物国家在目标地区的社会影响 `0.08`，其他国家影响对应降低并重新归一化。
- 组织社会活动：影响变化为 `投入 × 组织影响力 × 0.06`，要求 `regional_policy` 权限并消耗组织资源。
- 组织军事支援：只调用前线控制压力，要求 `regional_control_support` 权限并消耗组织资源；优先巩固薄弱己方前线，不修改社会影响。
- 社会影响变化不修改控制者、控制强度、争夺度或前线。
- 候选分数：熟悉 × 40 + 正向信任 × 25 + 正向好感 × 15 + 每个共同组织 15 + 声望 × 0.1；最低 20。
- 财富、声望、情报、盟友/敌对关系和职位继承比例按退出原因配置。财富从前人物扣除；声望和情报在继承者原值上增加；技能、资质和性格不复制。
- 敌对好感不高于 `-0.2` 时使用单独敌对关系比例，允许历史敌人继续存在。

全部数值位于 `data/balance/continuity_rules.json`。状态：M7 已验证，仍待长期玩法平衡。
