# 信息、知识、媒体与有限认知

状态：设计冻结 v1。调查、新闻、媒体、传闻、公开记录和机构内部信息必须建立在同一知识系统上。

## 1. 原则

权威世界状态与人物认知状态分离。玩家只能读取当前人物知道、能够合理估计或依法公开的信息；观察地图不等于读取全部真实数据。

知识检查与权限检查分开：人物可能知道某项机密但无权执行决定，也可能有职位权限但尚未收到最新地区报告。

## 2. 知识记录

```text
KnowledgeRecord
  knowledge_id
  knower_id: character | organization | institution
  subject_type
  subject_id
  fact_key
  value_kind: exact | range | category | claim | unknown
  perceived_value
  confidence: 0..100
  source_type
  source_id?
  acquired_hour
  observed_state_hour?
  public_scope
  secrecy_level
  expiry_rule
  contradiction_group?
  provenance_ids[]
```

`acquired_hour` 是得知时间，`observed_state_hour` 是信息所描述状态的时间；两者不同才能表达延迟和过时。

同一事实可存在相互矛盾的记录。系统不会悄悄把低可信传闻覆盖成世界真值；判断服务根据来源、时间和人物能力生成当前摘要。

## 3. 信息来源

| 来源 | 典型范围 | 优点 | 限制 |
| --- | --- | --- | --- |
| 本人经历 | 自己、住户、亲历互动 | 精确且及时 | 范围有限 |
| 当面交流 | 可接触人物 | 可追问 | 受信任、记忆和隐瞒影响 |
| 信件 | 已知地址与通信网 | 可跨地区保存 | 延迟、丢失、截取 |
| 电报 | 有设施的地区 | 快 | 昂贵、内容短、可审查 |
| 电话 | 连接地点 | 即时 | 覆盖有限、需双方接入 |
| 报纸 | 发行范围 | 可聚合公开事件 | 编辑立场、出版延迟 |
| 组织网络 | 成员与职位范围 | 专业或内部信息 | 按权限和层级过滤 |
| 公开记录 | 法院、政府、企业披露 | 可验证 | 可能滞后或不完整 |
| 调查 | 特定目标 | 提高精度 | 时间、费用、法律与风险 |
| 传闻 | 社交网络 | 传播快 | 可信度低、易变形 |

## 4. 人物信息可见度

本人可以查看精确健康、疲劳、压力、财务、技能、合同和已知关系。

查看其他人物默认只显示：姓名、公开职业/职位、已知地点、与玩家关系、估计生活状况、估计健康、已知公开立场和来源时间。

禁止天然显示：精确财富、精确健康、全部技能、完整关系、隐藏性格权重、内部倾向和未公开组织身份。

人物估计示例：

- 生活拮据（本月同事观察，可信度中等）
- 收入似乎稳定（旧雇佣记录，可能已过时）
- 健康欠佳（昨日亲眼所见，可信度较高）
- 情况未知

## 5. 组织、市场与国家信息

组织资源对公众显示“资金紧张/一般/充足/未知”；成员或财务职位根据权限看到更细账目。企业精确现金、库存和债务不因在地图上可见而公开。

市场公开层可以提供本地零售价格和可观察工资；批发库存、企业成本和跨区订单需要商业渠道或统计机构。

国家公开法律、政策、战争状态和正式公告可广泛观察；预算执行、军队位置和外交谈判按保密与职位权限限制。

## 6. 可信度与过时

当前可用可信度由以下结构计算：

```text
有效可信度 = 来源基础可信度
             + 直接观察/专业能力修正
             + 多来源相互印证
             - 时间衰减
             - 传播层数
             - 来源利益冲突
             - 已知操纵或审查
```

不同事实使用不同过时规则：人物当前位置数日即可能过时；职业可维持数月；出生年份通常不衰减；市场价格按市场周期迅速过时。

UI 必须显示“何时、从哪里知道”，不把估计伪装成精确值。

## 7. 信息传播

```text
世界事实/公开行为
→ 来源产生报道或观察
→ 通信网络传输
→ 媒体/组织筛选与改写
→ 人物获得 KnowledgeRecord
→ 影响其计划、关系与决定
```

传播速度与覆盖读取距离、道路、铁路、港口、邮政、电报、电话、识字率、语言、审查和组织网络。重大新闻也有传播延迟，不瞬时同步全球。

## 8. 新闻、通知、事件和数值结算

- 数值结算：普通工资、基础疲劳恢复、小价格变化；不主动通知。
- 通知：与当前人物直接相关的工资异常、解雇、明显关系变化或本地价格冲击。
- 事件：需要玩家决定的职位、借款、罢工、搬迁、帮助或危机。
- 世界新闻：战争、政权更替、重要法律、市场崩溃、重要人物死亡和技术突破。

新闻是一种可传播的信息产品，不是权威世界状态本身。错误报道、延迟更正和不同媒体立场可以存在，但首版只需要可信度与更正接口。

## 9. 调查

调查针对一个事实集合，而不是“揭示整个人物字典”。结果可以：

- 新增记录。
- 缩小估计范围。
- 提高或降低可信度。
- 发现冲突来源。
- 证明原信息已过时。

调查需要时间、费用、可达性、技能与合法手段。越权调查可能产生关系、法律和丑闻风险。

## 10. AI 认知

AI 使用自己的知识记录做决定，不直接读取超出其认知范围的玩家或世界真值。为控制性能，普通背景实体可保存聚合认知标签；活跃人物和关键机构保存具体记录。

AI 在信息不足时可以调查、询问、等待或采用风险折扣。错误信息可导致合理但非最优的计划，不能表现为随机失智。

## 11. UI 层级

- 扫视：估计分类、可信度图标、过时标记。
- 悬停：来源、获得时间、所描述时间和主要不确定性。
- 详情：来源链、冲突记录、调查入口和允许公开范围。
- QA：权威真值、全部知识记录、传播路径和计算权重。

正式 Release 只能访问前三层。

## 12. 存档与容量

保存当前仍有效或具有历史意义的知识记录、来源链和玩家最近查看的摘要；普通背景人口只保存聚合信息状态。过期且无事件、关系或决策引用的低价值记录可在月度边界压缩。

压缩必须保持玩家已经依据该信息作出的历史原因记录，不得让存档加载后出现“决定没有来源”。

## 13. 首版与保留接口

首版必须：人物/组织/市场基础知识记录、来源、时间、可信度、过时、人物估计显示、公开新闻和调查增量。

只保留接口：完整宣传模型、审查机关、密码通信、间谍网络、复杂报业市场、多语言翻译和大规模虚假信息行动。
