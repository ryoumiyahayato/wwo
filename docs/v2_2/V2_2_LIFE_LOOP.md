# V2.2 人物日常、正式时间与生活经济最小闭环

## 编码前架构审计

本记录在 V2.2 源码实现前建立。真实初始提交为
`272c22982679310564fd5adeaca184e177db6c34`。

### 复用

- `SimulationClock` 与 `SimulationClockConfig`：继续作为唯一权威时间源。评审场景只通过场景配置把时钟定位到
  `1900-03-12 05:00`，不改变全局 `1900-01-01 00:00` 默认值。
- `DeterministicRandomService`：保存种子和随机状态；V2.2 当前规则均为确定性条件判断，不引入第二套随机源。
- `StableIdService` 的稳定 ID 约定：人物、组织、地点、合同和活动使用可读稳定 ID。
- `GameSaveService`：在同一服务中增加 `v2_2_life_loop_1` schema 分支，继续使用 `.tmp`、校验、`.bak`
  和成功替换流程，不建立平行存档系统。
- V2.1.2-PERF 的 `PrototypeV2MapCanvas`、五档 LOD、统一空间索引、九个批绘图层、标签缓存和 Dirty
  Flag：V2.2 场景直接复用同一地图组件。
- `PrototypeV2Interface` 的四角入口、国家/机构/人物/组织视觉结构、RF 徽记、`✓`/`!`/`×`/锁定语义和
  Esc 逐层关闭逻辑。
- P0-R1 的人物、组织、关系、行动和存档实现继续保留；V2.2 不改变其产品规则和既有存档 schema。

### 修改

- `PrototypeV2Main`：允许 V2.2 子场景注入生活模拟绑定，并把空格键交给权威时钟命令。
- `PrototypeV2Interface`：静态人物、时间、经济、日程、关系、工会、通知和系统菜单字段在存在 V2.2
  绑定时改读实时只读状态；静态 V2.1.2 场景继续保留原有审查行为。
- `GameSaveService`：增加 V2.2 快照构建、验证、原子写入、读取和事务式恢复入口。
- 统一验证脚本：加入 V2.2 专项入口。

### 新增

- `V2LifeLoopConfig`：集中加载评审人物、地点、合同、价格、状态效果、NPC 规则和场景起点。
- `V2DateTime`：把权威整数小时转换为 Gregorian 日期、星期和幂等键。
- `V2ScheduleService`：保存活动、解决优先级与冲突、填补未来 48 小时日程。
- `V2EmploymentService`：记录合同义务、出勤、请假、缺勤、加班、周薪、月薪和津贴。
- `V2LedgerService` 与 `V2HouseholdService`：原子现金交易、库存、日消费、房租和欠款。
- `V2ConditionService`：疲劳、压力、健康、睡眠与食物缺口，并写入真实因果记录。
- `V2RelationshipProgressService` 与 `V2OrganizationActivityService`：实现联系让娜和工会例会的最小正式结果。
- `V2NotificationService`：区分个人提醒、组织信息和公开背景新闻，并聚合同类通知。
- `V2LifeLoopSimulation`：按固定顺序协调每个已完成小时，不持有第二个时钟。
- `V2LifeLoopUiBinding`：向自绘 UI 提供只读快照和结构化命令。

### 静态字段替换

V2.2 评审场景中的日期、星期、小时、暂停/倍率、当前/下一活动、地点、今日安排、现金、收入、支出、工资、
下一发薪、库存、下一房租、欠款、健康、疲劳、压力、就业风险、出勤、最近交易、关系变化、工会参与度和
通知全部来自生活模拟。V2.1.2 隔离场景仍可读取原静态审查数据。

### 不复用的旧逻辑

- `ActionService` 适合有工作量、效率和结果阈值的长期行动；小时级睡眠、通勤、购买和合同义务需要唯一时间
  占用及逐小时结算，直接套用会形成两套时间语义，因此只保留其“UI 提交命令、服务权威重验”的边界原则。
- `SocietySimulationService` 的日/月宏观 AI 与组织收入不是个人住户账本或劳动合同，不能作为 V2.2
  工资、消费和日程的权威来源。
- 静态世界动态里的市场新闻只作为明确的公开背景新闻；它不代表市场或供需定价已经运行。

### 存档版本

V2.2 使用 `schema_version = "v2_2_life_loop_1"` 和固定评审槽
`user://saves/v2_2_review_slot.json`。加载先解析和完整验证到临时状态，再同时替换时钟与生活模拟；版本错误、
损坏或引用错误不修改当前运行态。P0-R1 的 `save_version = 1` 保持不变。

### 地图性能隔离

生活服务不引用 `PrototypeV2MapCanvas`。UI 绑定只在时钟、人物或生活数据变化时局部请求界面重绘；每小时结算、
账本和通知不会设置地图的 geometry、label 或 camera Dirty Flag。地图只在观察人物切换或其地点显示确有变化
时接收最小状态。

### 当前最大风险

- 1280×720 下实时信息、活动安排和开发者信息可能过密：使用紧凑摘要、二级页面和最近五笔交易控制。
- 30 日逐小时运行可能积累大量活动、因果与通知：已完成活动、因果和通知都设置有界历史，未来日程按日增量
  补足而不每帧或每小时重建完整 48 小时。
- 周薪、日消费、房租和月薪在保存边界重复：所有周期结算都以持久化幂等键为最终判据。

## 范围边界

本轮固定生活价格、工资、房租和活动效果均为 Demo 平衡值，不是完整市场、企业现金流、正式行政模拟或历史
价格考据。正式空间、交通、通信延迟和有限认知留给 V2.3。
