# V2.2 实时 UI 绑定

V2.2 继续使用 V2.1.2-PERF 的四角界面和地图，不建立第二套 UI。`V2LifeLoopUiBinding` 负责把正式生活域状态转换为只读展示快照，并把玩家操作转换为领域命令。

## 原则

- UI 不持有权威人物状态。
- UI 不自行推进日期。
- UI 不直接修改现金、库存、工资或状态。
- UI 不直接写入账本。
- UI 不直接标记工资、房租或消费已结算。
- 所有命令返回结构化结果。
- UI 只在实际数据变化时刷新相关字段。
- 人物生活结算不得触发地图几何、标签或相机层重建。

## 时间入口

右上时间入口实时读取：

- 日期；
- 星期；
- 小时；
- 暂停状态；
- 当前倍率；
- 可用倍率。

可执行：

- 暂停；
- 1×；
- 2×；
- 4×；
- 8×。

安排活动、保存载入和开发者工具属于阻断面板。打开时自动暂停；关闭后恢复打开前的速度和暂停状态。

## 人物概览

实时字段包括：

- 当前活动；
- 下一活动；
- 当前地点；
- 活动结束时间；
- 职业；
- 雇主或所属机构；
- 工会或机构职位；
- 健康；
- 疲劳；
- 压力；
- 就业风险；
- 当前主要问题；
- 今日出勤；
- 当前生活计划。

状态符号：

- `✓`：正常或满足；
- `!`：需要注意；
- `×`：条件不足或严重风险；
- 锁：权限不足。

悬停说明必须来自正式因果记录，包含当前状态、主要原因、趋势、影响和建议。

## 生活与工作

实时字段包括：

- 个人现金；
- 本期收入；
- 本期支出；
- 周薪或月薪；
- 津贴；
- 下一发薪时间；
- 食品库存；
- 生活用品库存；
- 下一房租；
- 房租欠款；
- 最近交易；
- 工作义务、出勤、请假、缺勤和加班。

公务员个人现金和机构预算必须分区显示，不能共享字段或账本。

## 今日安排

人物中心显示连续活动段，而不是 24 个大型格子。

每段至少显示：

- 开始和结束；
- 活动名称；
- 地点；
- 来源；
- 状态。

来源：

- 合同；
- 默认日程；
- 玩家；
- NPC 规则；
- 系统。

状态：

- 已完成；
- 正在进行；
- 计划；
- 缺勤；
- 取消；
- 中断。

## 安排活动

正式支持：

- 加班；
- 授权无薪请假；
- 休息；
- 睡眠；
- 购买食品；
- 购买生活用品；
- 联系让娜；
- 参加工会例会；
- 制造无故缺勤评审场景；
- 取消尚未开始的玩家活动。

表单显示：

- 活动；
- 日期和时间；
- 持续时间；
- 地点；
- 现金成本；
- 时间成本；
- 预期效果；
- 工作义务影响；
- 条件符号；
- 冲突原因。

不显示伪概率。

## 关系与工会

联系让娜后实时更新：

- 熟悉度；
- 信任；
- 最近联系；
- 最近互动；
- 下一次可联系时间。

参加工会活动后实时更新：

- 工会参与度；
- 最近活动；
- 疲劳和压力影响；
- 组织通知。

V2.2 不实现复杂对话、关系 AI、工会选举或内部政治。

## 通知

通知分为：

- 个人提醒；
- 组织信息；
- 公开背景新闻。

正式生活通知包括工资、房租、物资不足、疲劳、睡眠、缺勤、购买、关系、工会、保存和载入。

同类通知应聚合。个人交易不得全部伪装为世界新闻。

旧的静态市场背景只可作为明确标记的公开背景新闻，不能暗示正式市场已经运行。

## 保存与系统工具

齿轮菜单中的正式功能：

- 保存 V2.2 评审进度；
- 载入最近 V2.2 评审存档。

设置和返回菜单仍可保持占位，必须明确说明。

## 评审与开发者模式

正常模式不显示身份切换条。

评审模式允许在皮埃尔和阿尔贝之间切换观察对象，但：

- 不重置时间；
- 不重置日程；
- 不停止另一人物；
- 只改变当前观察和操作对象。

开发者工具允许推进时间、制造工资和房租节点、修改现金和库存、设置状态、保存、载入和重置场景。内部幂等键不得进入普通玩家界面。

## 地图隔离

以下变化不得设置地图几何、标签或相机 Dirty Flag：

- 每小时结算；
- 现金变化；
- 账本追加；
- 通知追加；
- 工资到账；
- 房租支付；
- 状态变化；
- 关系变化。

只有人物位置、当前选择或实际覆盖层数据变化时，才允许更新相应轻量图层。
