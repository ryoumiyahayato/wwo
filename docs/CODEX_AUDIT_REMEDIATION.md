# Codex 审计问题对照与修复

日期：2026-07-13

审计基线提交：`9e92e77c422be782fd21baaa69ee7b41099ce8be`

本文件记录审计问题与默认分支修复的对应关系。源码完成不等于本机验收通过。

## P0：强类型编译阻断

- 问题：`MapControlService` 把 `ControlUnitData` 传给要求 `String` 的 `is_surrounded()`。
- 修复：改为传递 `unit.id`。
- 防回归：审计专项回归实例化战略地图及其强类型依赖；统一验证脚本扫描日志中的解析和加载错误，不再只看退出码。

## P0：成长与保证成功不可达

- 问题：学习行动固定提高调查技能，政治、行政和军事能力没有完整成长路径；充分准备仍不能到达保证线。
- 修复：
  - 正式行动面板增加技能选择。
  - 学习行动提高所选技能；旧行动未指定时默认原主技能。
  - 非学习行动按成功或失败提高其主要技能。
  - 主技能达到精通、准备和资金达到配置阈值时，给予明确的有效值加成。
- 防回归：`tests/codex_audit_regression.gd` 对普通模式与完整人口模式共 1000 种种子、全部行动定义验证保证成功可达性。

## P1：继承原因缺少权威状态约束

- 问题：健康年轻人物可任意声明死亡、退休、长期监禁或严重失势。
- 修复：
  - 退休检查年龄或权威继承状态。
  - 死亡检查健康、最大年龄或权威继承状态。
  - 长期监禁检查 `detained`。
  - 严重失势检查低声望、`disgraced` 或权威继承状态。
  - 自愿退出保持常规选项。
  - UI 只展示当前合法原因，服务端执行时再次校验。

## P1：NPC 新状态追溯旧时间

- 问题：日末新关系、职位、目标和权限可能被用于结算之前整天。
- 修复：
  - 日边界先记录新上下文和失效原因，但保留旧有效值与效率。
  - 先用旧上下文结算从上次更新时间到当前边界的区间。
  - 再从边界中断或采用新上下文，重算后续效率。
  - 日结算内部临时标记不得进入存档。

## P1：存档运行时不变量缺口

- 问题：可能恢复超过活跃上限、缺少 AI 状态、激活种子不完整或行动实例 ID 重复的存档。
- 修复：
  - 活跃人物数量不得超过配置上限。
  - 活跃/退出层级标记必须正确。
  - 激活种子必须完整覆盖所有人物，背景种子必须匹配记录。
  - 活跃 NPC 与 AI 状态必须一一对应，玩家不得有 AI 状态。
  - 玩家和所有 NPC 的进行中行动实例 ID 必须全局唯一。
  - NPC `current_action_id` 必须与行动定义一致。

## P1：1280×720 玩家旅程

- 问题：开始按钮位于滚动内容底部，自动旅程无法稳定滚动到可交互位置。
- 修复：正式开始按钮移动到滚动区外的面板底部。
- 当前旅程：`tests/p0_r1_player_journey_post_audit.gd` 通过正式 UI 执行关系、组织、职位、政策、合法退休继承及保存加载，并验证按钮在 1280×720 内可见。

## P2：干净检出测试不自举

- 问题：独立 `--script` 测试依赖已有 `.godot` 全局类缓存。
- 修复：最终入口先执行 Headless 编辑器导入，再运行测试。
- 最终命令：

```powershell
powershell -ExecutionPolicy Bypass -File 'D:\wwo\tools\run_post_audit_validation.ps1'
```

## P2：文档与性能证据分叉

- 当前结算顺序统一为：

```text
小时 → 日 → 月 → 到期队列事件 → 周/自动存档
```

- `README.md`、`ARCHITECTURE.md`、`DECISIONS.md`、`TEST_PLAN.md`、`ROADMAP.md`、`KNOWN_ISSUES.md` 和 `PERFORMANCE_BUDGET.md` 已区分历史证据、Codex 影子验证和当前未验证修改。
- 历史一年模拟约 `846 ms`；Codex 单点影子修复约 `1,916 ms`。二者都不能作为当前默认分支最终性能证据。

## 安全回归的备份快照比较

- 问题：JSON 往返可能把整数和浮点表现归一化，直接比较写入前内存字典会产生假失败。
- 修复：预期快照先经 `JSON.stringify` 与 `JSON.parse_string` 归一化，再与备份加载结果比较。
