# 数据模型

M2 已实现基础实体边界、稳定 ID、索引与加载验证；M3 加入正式世界，M4 扩展人物生成字段。后续领域行为不得绕开此处的验证规则。

## 稳定标识

国家、地区、控制单元、人口群体、人物、组织、关系和行动均使用稳定字符串 ID。格式为小写 `命名空间:slug`，例如 `country:example`；运行时生成 ID 使用八位递增序号，例如 `character:00000001`。不得使用对象内存地址、场景节点路径或未受控随机 UUID。

命名空间计数器可导出和恢复；加载器在全部集合间检查 ID 唯一性，并验证记录所属命名空间。

## 目标实体

- 国家：ID、名称、地区 ID 列表与公开状态。
- 行政地区：ID、名称、法理国家、人口群体、城市、资源、设施、组织与社会影响。
- 控制单元：ID、地区、显式 `grid_x/grid_y`、可选城市名、预存邻接、法理归属、军事控制者、控制强度、争夺度、驻守/敌方压力、社会支持、铁路与设施状态；五项连续状态按 `0..1` 验证。
- 人口群体：ID、地区、人数、阶层、职业类别、平均收入和教育、失业率、公开倾向与生活状态。
- 人物：基础身份、职业 ID/名称、公开职位、组织/关系 ID、六项隐藏潜质、气质权重、九项可见技能、已显现性格、真实/已知倾向、当前状态、随机模式、类别、困难开局标记、生成种子与 RNG 状态。
- 组织：ID、名称、类型、国家和可选地区、规模、资源、影响力、立场、领导人、成员、职位结构及与其他组织的关系。职位保存等级、槽位、权限和任职者稳定 ID。
- 关系：ID、双方人物、熟悉、信任、好感、利益关联、公开性和最后互动小时；只保存实际存在的关系。
- 行动定义：ID、名称、类别、工作量、每小时基础进度、技能、资质/组织/关系/资金/准备/状态权重、基础阻力、职位要求、中断条件、两个成功阈值与结果。M5 正式世界包含八类定义。
- 行动实例：稳定 ID、定义/人物/目标 ID、状态、开始/上次结算/完成小时、已完成与总工作量、当前效率、预计完成小时、依赖上下文、内部有效值、公开定性把握、结果及一次性应用标记。

所有模型都是无场景依赖的强类型 `RefCounted`，并通过 `to_dict()` 返回深拷贝的可序列化字段。

## 所有权边界

控制单元必须分别保存法理归属、实际军事控制和社会/政治影响，不使用单一 `owner` 字段。

## 存档映射

未来存档只保存可序列化数据、稳定 ID、配置版本、随机状态与权威游戏时间。初始格式草案见 `docs/SAVE_FORMAT.md`。

## 加载验证

核心 JSON 当前使用 `schema_version = 1`，顶层包含 `countries`、`regions`、`control_units`、`population_groups`、`characters`、`organizations`、`relationships` 和 `actions` 八个数组。加载器验证：

- 必填字段、类型、非空约束、数字范围和数组重复值。
- ID 格式、集合命名空间和全局唯一性。
- 国家—地区、地区—人口/组织、控制单元—地区/国家/邻居、人物—国家/地区/组织/关系、组织—领导/成员和关系—双方人物的引用。
- 关系双方不能相同，控制单元不能与自身相邻，必然成功阈值不得小于普通成功阈值。
- 控制单元网格坐标全局唯一，邻接必须双向，铁路只连接相邻单元，城市必须在所属地区声明。

任何错误都会返回完整错误列表，不提供部分 `CoreDataSet`。

## M3 正式世界

- 国家：洛岚联邦、维斯塔联盟。
- 行政地区：每国 4 个，共 8 个。
- 控制单元：`10×8` 正交网格，共 80 个；每地区 10 个。
- 人口群体：每地区一个汇总群体，共 8 个。
- 城市：晨港、铁川、星城、赤原。
- 铁路：23 条唯一相邻连接，组成一条东西干线和两条南北支线。
- 初始边境：第 4/5 列之间，共 8 条自动前线边。

这些是正式 Demo 架空地理数据，不预置人物或 M6 组织内容；M4 玩家人物在运行时确定性生成。

## M4 人物信息边界

`CharacterData.to_dict()` 返回完整可序列化数据；`to_public_dict()` 只返回正式可见身份、技能、已显现性格、已知倾向和当前状态，不含隐藏潜质、气质权重、真实倾向、生成种子或 RNG 状态。UI 默认只读取公开层，开发者面板必须显式开启。

## M5 行动状态

行动状态为 `active`、`paused`、`completed`、`cancelled` 或 `interrupted`。终止状态不可再次推进或重复应用结果。会话只保存一个玩家主要行动；M8 已把实例全部字段映射到版本 1 存档，而不是只保存预计完成小时。

## M8 持久状态

存档不序列化节点或内存地址。人物按背景、活跃、退出三层保存；组织保存成员和职位槽位；关系保存稀疏记录及人物对索引可重建字段；AI 只保存活跃 NPC 状态。静态国家、法理、邻接、铁路和行动定义继续来自版本化核心数据，存档只覆盖可变运行字段。

## M6 人物层级、组织和关系

背景人物使用 `BackgroundCharacterData`，只保存稳定 ID、公开身份、职业/职位、组织/关系 ID、已显现性格、简化状态和稳定激活种子；不保存完整技能、隐藏资质、真实倾向或 AI 状态。活跃人物继续使用完整 `CharacterData`。升降级移动同一 ID 的记录，不创建同名新人物。

关系仍使用 `RelationshipData`，以排序人物对建立唯一运行索引，熟悉度为 `0..1`，信任和好感为 `-1..1`。只有发生实际接触时才创建记录。组织运行索引保存人物在各组织的当前职位；职位权限是 M5 受限行动的唯一权限来源。

## M7 地区影响与人物连续性

地区社会影响继续存于 `RegionData.social_influence`，按国家 ID 归一化；军事状态继续存于控制单元。两类服务没有共享 `owner` 或隐式转换。

退出人物由 `ExitedCharacterRecord` 保存完整人物、退出原因/小时和继承者 ID，并从活跃/背景索引移动到历史索引。`SuccessionCandidateData` 保存候选稳定 ID、来源角色、关系 ID、共同组织和可解释分数。继承结果只记录实际转移的财富、声望、情报、职位、关系及敌对关系数量，不覆盖继承者自身技能、资质、性格或经历。
