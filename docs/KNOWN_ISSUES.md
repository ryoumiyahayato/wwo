# 已知问题

## P0 可玩性复审

- M0 至 M9 的历史实现、自动检查和 Windows 11 导出记录保留，但人工验收确认普通玩家闭环尚未达到 Demo 完成标准，现进入 `P0-R1` 修复阶段。
- `P0-R1` 已完成源码侧第一轮补全：战略地图玩家入口可换行；长期行动条件改由人物财富、技能、关系、组织职位与目标状态自动计算；正式社会面板隐藏直接加入、改职位和直接建关系入口；活跃 NPC 开始执行有界日常行动和月度地区影响；战略地图新增普通玩家“保存游戏”；人物页面返回地图时复用同一时钟、地图、社会和行动会话。
- 信息面板的直接压力和模拟易手、人物隐藏数据与倾向事件、社会系统直接修改按钮仍保留为开发验证能力，但在普通模式下不可见；开启开发模式后才显示。
- 网页端无法运行本机 Godot。上述源码改动仍需在本地执行 headless 测试、1280×720 Compatibility 交互测试、存档往返和重新导出 EXE；未验证前不得宣称 `P0-R1` 或 P0 Demo 已完成。
- 现有 564 项测试包含若干直接调用服务或程序化触发隐藏控件的历史测试；它们用于保持底层回归，但不能替代新的真实玩家旅程验收。

## 当前限制

- 新游戏已进入人物创建流程；存在手动档时加载游戏可用，设置仍处于禁用状态。
- M3 已有正式双国战略地图，但地理为程序绘制的正交架空占位结构，不是最终美术或真实地图。
- P0-R1 的行动费用和自动条件换算是首版可玩性参数，尚未经过平衡测试；当前行动仍以定性把握和阈值结算，尚未加入中间区间的确定性随机判定。
- 活跃 NPC 的执行层目前只覆盖休息、工作、学习、加入组织、维持既有关系，以及具备职位权限时的月度地区社会或军事支援；它是可观察世界变化的最小实现，不代表完整人物 AI。
- AI 不会为每个 NPC 建立完整长期行动实例，也不会运行复杂战争、经济或外交系统；这些不属于本轮 P0-R1 最小修复范围。
- 人物退出与继承仍要求先通过长期行动形成足够强的关系或共同组织，继承权重和比例是首版值。
- 背景人物降级保留稳定 ID、公开经历、组织和关系 ID；为维持轻量预算，不保留完整技能训练过程，重新升级由稳定激活种子补全内部字段。该限制同样适用于被选作继承者前的背景人物。
- 版本 1 存档可跨进程恢复地图、组织、关系、AI、退出历史、事件队列、行动进度和稳定 ID 计数器；P0-R1 新增的同进程页面切换连续性仍需本机验证。
- 安全替换只在当前 Windows 11 开发环境验证；Linux、macOS 和 Windows 10 的文件替换语义尚未真机测试。
- 自动存档在权威周边界同步执行并使用一个槽位；重新导出的 P0-R1 产物仍需复测。
- Windows Desktop 旧版系统原型曾导出为嵌入 PCK 的 x86-64 单 EXE；P0-R1 修改后尚未重新导出。EXE 未签名，首次启动可能显示 Windows 未知发布者提示。
- Windows 10 为目标兼容平台，尚未实机验证。Linux 和 macOS 仅有导出预设，未实际导出或测试，不宣称兼容性已通过。
- 开发者面板在 1280×720 下信息较密，长命令帮助文本可能被裁切。

当前准确状态是：Windows 11 可执行系统原型的历史产物存在，`P0-R1` 源码修复已推进到本机验证前阶段；只有完成本地自动测试、真实玩家旅程、保存加载、页面切换连续性和重新导出验收后，才能重新判断 P0 是否完成。
