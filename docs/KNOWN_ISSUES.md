# 已知问题

## P0 可玩性复审

- M0 至 M9 的历史实现、自动检查和 Windows 11 导出记录保留，但当前默认分支仍处于 `P0-R1` 修复与复验阶段，不满足发布条件。
- 2026-07-13 Codex 使用精确 Godot `4.6.3.stable.official.7d41c59c4` 审计了提交 `9e92e77c422be782fd21baaa69ee7b41099ce8be`。原始 Headless 进程虽然返回 `0`，日志包含强类型解析错误，因此实际不通过。
- 审计后的默认分支已修复该编译阻断，并加入自举验证脚本：先执行 Headless 编辑器导入生成全局类缓存，再运行全部测试，同时扫描解析和脚本加载错误。
- 核心成长闭环已补充：正式学习可选择任意人物技能，非学习行动产生主技能实践成长；主技能达到精通、准备和资金均达到配置阈值时，公式提供明确的保证成功上界。
- 继承退出原因已增加权威约束：退休、死亡、长期监禁和严重失势必须由年龄、健康、拘禁、声望或明确状态证明；自愿退出保持可用。
- NPC 日结算已改为先按旧效率结算已过去区间，再从日边界应用新目标、权限和状态；不再使用日末新状态追溯整天。
- 存档恢复已增加活跃人物上限、人物层级、激活种子完整性、活跃 NPC 的 AI 全覆盖，以及玩家/NPC 行动实例 ID 全局唯一性校验。
- 1280×720 行动面板的正式开始按钮已固定在滚动区外；备份恢复测试改为按 JSON 语义归一化比较。
- 新增 `tests/codex_audit_regression.gd`，覆盖上述审计项并对普通与完整人口模式共 1000 种种子验证核心行动保证成功可达性。

## 当前设计限制

- 设置界面仍处于禁用状态；地图仍是程序绘制的正交架空占位地理。
- 主动投入只使用人物财富；情报点仍是自动准备因素，不是单独可消耗资源。
- 技能成长、精通保证、组织收入、生命周期和控制压力权重均为首版值，需要长期模拟和平衡测试。
- 活跃 NPC 只从有限规则候选中选择，不运行复杂外交、市场或完整战争计划。
- NPC 的月度组织地区行动仍是组织层级结算，不为每次组织决策创建人物行动实例。
- 玩家满足退休或死亡条件时会设置继承要求，但尚未增加强制遮罩或自动弹出继承界面。
- 控制压力同时读取地区 `social_influence` 和单元 `social_support`，两层权重尚未经过长期平衡测试。
- 继承最多新增两项组织身份并保留继承者原有身份，具体继承比例仍是首版值。
- 背景人物保存完整成长核心，新增 NPC 行动和资金审计后，存档体积必须重新测量。
- 旧版本 1 存档缺少部分新字段时的兼容路径仍需真实旧档往返验证。
- Windows 10、Linux 和 macOS 尚无当前版本实机证据；当前 Windows EXE 也尚未重新导出。
- 开发者面板在 1280×720 下信息较密，长文本仍可能裁切。

当前准确状态是：审计指出的确定性代码问题已在源码和专项回归层面逐项处理，但当前默认分支尚未获得完整本机验证。只有统一自动验收、GUI 玩家旅程、保存加载、30 日与一年模拟、性能和重新导出全部通过后，才能重新判断 P0-R1 是否健全。
