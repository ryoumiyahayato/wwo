# 已知问题

## P0 可玩性复审

- M0 至 M9 的历史实现、自动检查和 Windows 11 导出记录保留，但人工验收确认普通玩家闭环尚未达到 Demo 完成标准，现处于 `P0-R1` 修复与复验阶段。
- `P0-R1` 已完成源码侧玩家入口补全：战略地图入口可换行；行动条件来自权威人物、组织、关系、资源和目标状态；正式社会面板隐藏直接修改入口；战略地图提供普通玩家保存；页面间复用同一世界会话。
- 第一批状态一致性修复已经合并到默认分支：背景人物降级保存完整成长核心；继承先释放旧玩家槽位并在失败时恢复人物、组织、关系和 AI；加入组织与入口职位分配原子化；行动每个权威小时重读真实依赖；领域写入失败撤销成功奖励；进行中行动存档对照权威世界验证。
- 模拟质量修复已经直接写入默认分支：背景人口按国家人口、人口群体和职业结构抽样；玩家可选择额外财富投入；NPC 使用同一行动实例、费用、工作量和结果规则；组织获得月度收入；AI 可从空关系网络建立第一条关系；人物自然增龄、健康衰退、退休和死亡；政策与军事支援检查国家和组织辖区；铁路、地区社会影响、控制单元本地支持、多方向进攻和包围参与控制压力；继承合并继承者原有关系并限制新增组织身份；人物页面继续驱动同一权威时钟。
- 信息面板的直接压力和模拟易手、人物隐藏数据与倾向事件、社会系统直接修改按钮仍保留为开发验证能力，但普通模式不可见。
- 当前连接环境无法运行指定 Godot 4.6.3，仓库也没有 GitHub Actions。上述源码改动仍须在本地执行原有回归、P0-R1 逻辑与玩家旅程、状态一致性、模拟质量回归、1280×720 Compatibility 交互测试、存档往返、一年模拟和重新导出 EXE；未验证前不得宣称 P0-R1 或 P0 Demo 已完成。
- 现有 564 项历史测试包含直接调用服务或程序化触发隐藏控件的路径，只能作为底层回归记录，不能替代当前真实玩家和模拟质量验收。

## 当前限制

- 新游戏已进入人物创建流程；存在手动档时加载游戏可用，设置仍处于禁用状态。
- 地图仍是程序绘制的正交架空占位地理，不是最终美术或真实地图。
- 主动投入当前只使用人物财富；情报点作为自动准备因素，不作为单独可消耗资源。投入上限与公式均是首版平衡值，尚未经过人工平衡测试。
- 活跃 NPC 现在保存并推进真实长期行动实例，但仍只从有限规则候选中选择，不运行复杂战争计划、外交、市场或完整经济系统。
- NPC 的月度组织地区行动仍是组织层级结算，直接消耗组织资源，不为每次组织决策建立人物行动实例。
- 组织月度收入、人物退休年龄、健康衰退和控制压力倍率均为首版参数，需要在 30 日与一年模拟中验证不会导致资源无限积累、领导频繁更替或前线过快崩溃。
- 玩家达到退休或死亡条件时设置 `succession_required` 状态，但尚未增加强制遮罩或自动弹出继承界面；玩家仍需从社会系统进入继承流程。
- 控制压力已经同时读取地区 `social_influence` 和控制单元 `social_support`；两层支持的权重是首版参数，尚未经过长期平衡测试。
- 继承最多新增两项组织身份，并保留继承者原有身份；具体权重和职位继承比例仍是首版值。
- 背景人物保存技能、资质、真实倾向和随机核心以维持连续性，存档体积需要重新测量。
- 版本 1 存档兼容缺少背景持久核心和完整资金审计字段的旧存档；新增 NPC 进行中行动、生命周期状态和投入字段仍需实际旧档往返验证。
- 安全替换只在历史 Windows 11 开发环境验证；Linux、macOS 和 Windows 10 的文件替换语义尚未真机测试。
- 自动存档在权威周边界同步执行并使用一个槽位；重新导出的 P0-R1 产物仍需复测。
- Windows Desktop 旧版系统原型曾导出为嵌入 PCK 的 x86-64 单 EXE；当前默认分支修改后尚未重新导出。EXE 未签名，首次启动可能显示 Windows 未知发布者提示。
- Windows 10 为目标兼容平台，尚未实机验证。Linux 和 macOS 仅有导出预设，不宣称已通过。
- 开发者面板在 1280×720 下信息较密，长文本可能被裁切。

当前准确状态是：状态一致性和模拟质量源码修复均已位于默认分支，并配有专用回归脚本；但尚未在指定 Godot 4.6.3 本机环境执行。只有全部自动测试、玩家旅程、保存加载、页面连续性、30 日与一年模拟、性能和重新导出验收通过后，才能重新判断 P0 是否完成。
