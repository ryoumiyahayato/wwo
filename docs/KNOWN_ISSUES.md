# 已知问题

## P0 可玩性复审

- M0 至 M9 的历史实现、自动检查和 Windows 11 导出记录保留，但人工验收确认普通玩家闭环尚未达到 Demo 完成标准，现进入 `P0-R1` 修复阶段。
- `P0-R1` 第一项已在源码中处理：战略地图顶栏改为可换行布局，并将“人物信息 / 长期行动 / 社会系统 / 开发工具 / 返回主菜单”改为明确、固定的玩家入口，目标是在 1280×720 下不再被时间控件挤出窗口。
- 网页端无法运行本机 Godot；上述布局仍需在本地拉取后执行 headless 测试、1280×720 实际渲染和鼠标点击验收，未验证前不得宣称此项最终通过。

## 当前限制

- 新游戏已进入人物创建流程；存在手动档时加载游戏可用，设置仍处于禁用状态。
- M3 已有正式双国战略地图，但地理为程序绘制的正交架空占位结构，不是最终美术或真实地图。
- 信息面板的直接压力和易手按钮仍是 M3 开发验证入口；M5 已提供受行动权限输入约束的控制支援，但正式职位来源属于 M6。
- M4 姓名池、职业和人口抽样权重是小规模架空 Demo 配置，不代表历史人口统计。
- 人物面板的倾向事件按钮仍是 M4 开发验证入口，不属于行动列表。
- M5 行动面板现在读取 M6 当前组织职位权限；社会面板中的直接加入和职位调整仍是 M6 机制验证入口，尚未完全由行动结果驱动。
- M7 已将政策行动连接到地区社会影响，并保留控制支援的军事通道；建立关系、加入组织和争取职位行动的目标选择仍主要通过社会面板完成。
- 背景人物降级保留稳定 ID、公开经历、组织和关系 ID；为维持轻量预算，不保留完整技能训练过程，重新升级由稳定激活种子补全内部字段。该限制同样适用于被选作继承者前的背景人物。
- 版本 1 存档可跨进程恢复继承后的地图、组织、关系、AI、退出历史、事件队列、行动进度和稳定 ID 计数器。
- 安全替换只在当前 Windows 11 开发环境验证；Linux、macOS 和 Windows 10 的文件替换语义尚未真机测试。
- 自动存档在权威周边界同步执行并使用一个槽位；一年回归的 52 次写入均低于单次 100 毫秒预算，但最终导出产物仍需复测。
- Windows Desktop Release 已导出为嵌入 PCK 的 x86-64 单 EXE，并在 Windows 11 完成交互冒烟。未签名，首次启动可能显示 Windows 的未知发布者提示；本项目未测试或绕过该提示。
- Windows 10 为目标兼容平台，尚未实机验证。Linux 和 macOS 仅有导出预设，未实际导出或测试，不宣称兼容性已通过。
- 开发者面板在 1280×720 下信息较密，长命令帮助文本可能被裁切；手动保存等核心按钮仍可用且面板默认隐藏。
- 当前已达到 80 地图单元、120 世界人物和最多 20 活跃人物的目标规模；一年稳定性回归已通过，但当前 AI 只完成候选评分和状态记录，尚不能据此证明世界具有可观察的自主变化。

M9 的 564 项自动检查、headless 启动、Release 导出和 Windows 11 导出 EXE 冒烟证明底层系统可以运行和导出，但不能证明普通玩家闭环完成。当前准确状态是：Windows 11 可执行系统原型已完成，`P0-R1` 可玩性修复进行中，Windows 10 外部实机验证仍待补。
