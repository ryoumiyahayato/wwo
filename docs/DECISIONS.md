# 架构决定

本文件只保留当前有效决定。后续决定覆盖早期描述时，早期描述不得继续作为并列权威。

## 2026-07-19 · 正式世界地图权威纠正

- 决定：`data/world_map/*` 是当前正式国家、地区、行政几何、城市、港口和跨地区交通数据来源；`scripts/world_map/*` 是正式地图加载、投影、LOD、空间索引、批绘和对象命中的实现入口。
- 决定：普通启动恢复为 `scenes/v2_3/v2_3_life_loop_menu.tscn`。普通运行不得进入旧 10×8 架空网格 Alpha。
- 决定：`data/world/demo_world.json` 中的两国、八区、80 单元正交网格只保留为历史技术、性能、存档迁移和战争控制夹具，不再是 Alpha 或正式产品的国家、地区、城市与地图权威。
- 决定：洛岚联邦、维斯塔联盟、晨港、铁川、星城和赤原不再作为正式世界内容。相关 `data/alpha/*.json`、`scripts/alpha/*` 和测试只能作为待迁移实现候选或明确标记的旧架空夹具使用。
- 决定：2026-07-19 将旧网格“升格为有限正式世界”、将真实世界地图延期到 Alpha 之后、废弃 `data/world_map/*` 正式地位的决定全部废止。
- 原因：此前已经投入的新世界地图、真实世界地理数据、投影、缩放、LOD、空间索引和交互实现就是为了替换旧网格。复用旧系统不等于复活旧世界内容。

## 正式世界与多精度原则

- 正式世界方向为 1900 年真实世界与多精度模拟。
- 当前 `data/world_map/*` 使用现代 Natural Earth 几何作为已知的过渡几何，国家主题名与法国详细地区用于地图和交互实现；它不是完整历史边界完成声明。
- 玩家当前活动地区可以使用详细人物、地点、日程、企业和组织模拟；重要关联地区使用简化市场、企业、组织和事件；远方地区使用聚合人口、经济、政策、贸易与战争状态。
- 所有精度层必须引用同一国家、地区、城市、地点和交通 ID 体系。不得让详细层和远方层分别使用架空网格与真实世界两套权威。
- 旧网格可继续用于可重复的性能与迁移测试，但测试入口必须与普通入口隔离，并明确显示“历史架空网格夹具”。

## Alpha 系统处置

- `scripts/alpha/*` 中的账本、资产、合同、债务、劳动、企业、人物发展、组织政治、AI、存档和测试代码暂不删除。
- 这些代码只有在完成以下条件后才能重新标记为正式 Alpha：
  1. 国家、地区、城市、地点和交通全部迁移到 `data/world_map/*` 权威；
  2. 普通 UI 能实际完成主要生命周期，而不是只由场景脚本直接调用服务；
  3. 玩家与 AI 使用同一正式入口；
  4. 存档迁移不把架空国家和地区静默写入正式存档；
  5. 自动测试覆盖普通入口与真实地图引用。
- 在上述迁移完成前，`prototype_0_001_alpha_1` 只能视为旧架空网格技术夹具 Schema，不得用其性能和测试结果宣称正式 Alpha 验收通过。

## 2026-07-19 · 首个正式地图服务迁移：个人借款

- 决定：个人借款不直接复用 `AlphaEconomyService`，因为该服务拥有独立 `AlphaLedgerService`，会与现有 V2.2 住户现金形成第二权威。
- 决定：正式入口使用 `V23FinanceService` 保存放贷方资金、申请、审查、借款合同、利息、期限、逾期和违约；人物现金仍只能由 `V2LedgerService` 修改。
- 决定：里尔信贷数据位于 `data/v2_3/lille_finance.json`，放贷方引用正式地点 `location_lille_centre`。任何正式金融状态不得引用 `loran_*`、`vesta_*` 或 `control:*`。
- 决定：申请行为始终客观存在，但要求人物实际位于办理地点且放贷方正在营业。劳动收入、现金和已有债务改变审查结果，不以技能等级隐藏整个功能。
- 决定：申请提交不直接产生现金；时间推进后形成条件或拒绝；接受条件后才通过住户账本放款。还款同样通过住户账本，重复操作由幂等键阻止。
- 决定：正式财务面板只显示现金、已知债务、地点、营业状态、产品条款、申请状态和合同状态，不推荐借款或还款路线。
- 决定：`formal_finance_state` 作为 V2.3 存档的可选扩展字段保存。旧 V2.3 存档没有该字段时，载入为空金融状态，不把旧架空债务迁入正式世界。

## 旧系统复用原则

优先复用且不得无理由重写：

- `SimulationClock` 与小时、日、周、月边界；
- `V2ScheduleService` 与工作、睡眠、生活安排；
- 地点、路线、旅行、消息、知识、约见、介绍和六维关系；
- 人物生成、隐藏资质、技能、状态和分层名册；
- 组织成员、职位和权限索引；
- 确定性随机、存档安全替换、幂等和自动测试；
- 已有账本原子批次与正式资金流规则。

复用这些系统时必须改造其国家、地区、地点和交通引用，使其读取正式世界地图。不得以稳定 ID、测试数量或性能较好为理由继续把旧网格作为产品世界。

## 唯一权威边界

| 对象 | 当前权威 | 禁止的第二权威 |
|---|---|---|
| 时间 | `SimulationClock` | UI 或 Alpha 自建时钟 |
| 正式地图几何与国家目录 | `data/world_map/*` + `scripts/world_map/*` | `AlphaWorldService` 生成的 10×8 网格 |
| 人物 | `CharacterData` / `CharacterRosterService` | Alpha 独立人物副本 |
| 地点与旅行 | `SpatialLocationService` / `TravelExecutionService` | UI 直接改当前位置 |
| 关系与知识 | V2.3 正式服务 | UI 或 AI 读取全局真相 |
| 组织、成员、职位 | `OrganizationService` | 企业或政治各自复制成员职位 |
| 现金与资金流 | `V2LedgerService` + `V2HouseholdService` | `AlphaLedgerService` 或合同、债务分别改现金 |
| 正式个人债务 | `V23FinanceService` 合同余额 | 住户字段自行累计或旧架空债务副本 |
| RNG | `DeterministicRandomService` | 临时全局随机 |

## 数据流与实现边界

依赖方向固定为：

```text
数据配置 → 核心模型与服务 → 模拟组合根 → 只读状态与事件 → UI
```

- UI 不保存模拟公式和正式状态。
- 地图图层只投影正式世界数据，不保存第二份地图状态。
- 正式数据加载前验证结构、类型、范围、稳定 ID 和跨实体引用。
- 业务逻辑不得在 `_process()` 或 `_physics_process()` 中全量遍历世界。
- 周期结算使用到期队列、活跃索引、批处理和脏标记。

## 人物与具身化权力

- 玩家始终控制具体人物。
- 人物可以亲自处理、请求协助、购买服务、建立合作、委托代理或通过职位发布正式指令。
- 资质、技能、经验、资格、关系、财富和职位改变信息、方法、质量、规模、依赖和后果，不以 UI 等级解锁整个玩法领域。
- 背景人物不创建可见节点；活跃人物上限保持有界；关系按实际接触稀疏创建。

## 时间、存档与验证

- 指定引擎为 Godot `4.6.3.stable.official.7d41c59c4`，Compatibility 渲染器。
- 权威时间使用自 1900-01-01 起的整数游戏小时，不进行全局分钟迁移。
- 所有业务随机通过确定性随机服务，存档保存种子和 RNG 状态。
- 存档写入临时文件，校验后安全替换；加载先恢复到临时组合根，完整验证后再替换当前会话。
- Godot 退出码为 0 不足以判定通过；日志中的解析、类型、脚本加载和断言错误必须使验证失败。
- 性能与发布声明只能引用同一提交、同一引擎版本和完整命令产生的证据。

## 当前发布状态

- 正式世界地图入口已恢复。
- 旧架空 Alpha 已退出普通启动并被隔离为内部回归夹具。
- 个人借款与还款是首个迁入正式 V2.3 地图、普通 UI 和正式存档的旧 Alpha 能力领域。
- 企业、通用资产、完整合同、组织政治、人物发展和 Alpha AI 仍未迁入正式入口。
- 当前不得使用“完整 Alpha 功能与性能验收通过”作为项目状态。
- 下一次正式 Alpha 验收必须在更多主要生命周期迁入真实地图权威、普通玩家入口和跨系统运行后重新进行。
