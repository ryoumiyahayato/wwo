# 路线图

一次只完成最早未完成的里程碑。长期愿景不扩大当前里程碑范围。

## M0：项目初始化和文档

- 目标：建立可打开、可显示菜单、可自动检查的最小 Godot 项目。
- 范围：项目配置、Compatibility、目录与 Git、核心文档、主菜单、日志、测试运行器。
- 验收：项目可打开；菜单可显示；测试命令和 headless 启动成功；文档明确 M1。
- 测试：项目设置、核心文档、日志、场景加载与真实启动。
- 当前状态：已完成（2026-07-11）；25/25 自动检查通过，headless 与 Compatibility 窗口启动退出码均为 0，1280×720 渲染首帧检查正常。

## M1：游戏时钟和调度器

- 目标：建立与渲染帧率无关的权威离散游戏时间。
- 范围：1900 起始时间、暂停、1/2/4/8 倍速、单步、周期事件和事件队列。
- 验收：UI 显示时间；不同帧率与相同操作产生一致结果。
- 测试：确定性时间、速度、暂停、单步和小时/日/周/月边界。
- 当前状态：已完成（2026-07-11）；项目累计 64/64 自动检查通过，headless 启动和 Compatibility 视觉检查正常。
- 已知问题：时钟和队列当前只存在于运行内存；持久化属于 M8，完整世界负载性能属于 M9。

- [x] 实现 1900 年开始时间
- [x] 实现暂停和 1/2/4/8 倍速度
- [x] 实现单步 1 小时
- [x] 实现小时、日、周、月事件
- [x] 实现事件队列
- [x] UI 显示日期和时间

- [x] 编写确定性时间测试

## M2：核心数据和随机种子

- 目标：建立稳定实体模型与统一确定性随机服务。
- 范围：稳定 ID、国家、地区、控制单元、人物、组织、关系、行动和数据验证。
- 验收：相同种子及操作顺序结果一致，数据错误可报告。
- 测试：固定种子、ID、序列化数据和加载验证。
- 当前状态：已完成（2026-07-11）；项目累计 104/104 自动检查通过，随机与数据加载日志干净。
- 已知问题：M2 fixture 不是正式世界；随机状态和 ID 计数器的存档字段留待 M8 固定。

- [x] 稳定 ID 与核心数据模型
- [x] 统一随机服务和固定种子测试
- [x] 数据加载验证

## M3：二维地图和动态控制区

- 目标：显示并更新双层地图及自动前线。
- 范围：2 国、8 地区、64 至 96 单元、拖放缩放、所有权分类、争夺、铁路和前线。
- 验收：控制变化正确反映于地图与前线。
- 测试：边界、孤立、包围、易手和前线消失。
- 当前状态：已完成（2026-07-12）；项目累计 158/158 自动检查通过，正式世界数据、headless 日志和 Compatibility 视觉检查正常。
- 已知问题：地图为架空正交占位地理；压力按钮是机制验证入口，不代表人物权限；状态持久化留待 M8。

- [x] 世界数据与可交互地图
- [x] 动态控制、铁路和前线
- [x] 地图边界测试

## M4：随机人物和人物信息

- 目标：从玩家选择国家按三种模式生成并展示人物。
- 范围：资质、技能、性格、倾向、状态、人物面板和隐藏信息开发视图。
- 验收：同种子生成相同人物，正式 UI 不泄露隐藏信息。
- 测试：生成确定性、年龄规则、倾向与信息可见性。
- 当前状态：已完成（2026-07-12）；项目累计 201/201 自动检查通过，同种子人物、三类随机、年龄规则、倾向变化和信息边界验证正常。
- 已知问题：姓名池和职业权重是 Demo 级架空样本；跨场景人物只保存在内存，持久化留待 M8。

- [x] 国家和随机模式选择
- [x] 人物生成与信息面板
- [x] 确定性和隐藏信息测试

## M5：行动和必然成功机制

- 目标：实现八类按时间推进、可准备并可中断的行动。
- 范围：工作量、效率因素、结果描述、必然成功和结果应用。
- 验收：行动与帧率无关，常规行动可通过准备达到必然成功。
- 测试：进度、暂停、取消、中断、完成、阈值和帧率一致性。
- 当前状态：底层服务和原有自动检查已完成；普通玩家入口与真实可操作性在 `P0-R1` 重新验收。
- 已知问题：公式权重是首版平衡值；组织、关系和正式职位结果需由 M6 实体系统接续，行动持久化留待 M8。

- [x] 八类基础行动与进度
- [x] 支持、准备、状态和阈值
- [x] 行动确定性测试
- [ ] 普通玩家在 1280×720 导出版本中可发现、打开并操作八类行动

## M6：组织、职位、关系和简化 AI

- 目标：让权力来源与活跃人物行为可运行。
- 范围：四类组织、职位权限、按需关系、人物升降级、每日和长期 AI。
- 验收：背景人物不运行完整 AI，活跃人物不超过 20。
- 测试：组织权限、关系、升降级和性能验证。
- 当前状态：底层服务和原有自动检查已完成；正式玩法入口、AI 执行层与自主世界变化在 `P0-R1` 重新验收。
- 已知问题：AI 与职位评分是首版规则；地区影响、人物退出和继承属于 M7，社会状态持久化属于 M8。

- [x] 组织、职位和关系

## M7：地区影响与人物继承

- 已知问题：继承比例是首版规则；跨系统行动目标选择仍较简化，完整状态持久化属于 M8。
