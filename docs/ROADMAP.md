# 路线图

一次只完成最早未完成的里程碑。源码完成与本机验收分开记录；没有测试证据时不得把里程碑标记为完成。

## M0：项目初始化和文档

- 目标：建立可打开、可显示菜单、可自动检查的最小 Godot 项目。
- 历史状态：已完成（2026-07-11）；25/25 自动检查和当时的启动检查通过。
- [x] 项目结构、Compatibility、主菜单、日志和自包含测试运行器。

## M1：游戏时钟和调度器

- 目标：建立与渲染帧率无关的权威离散游戏时间。
- 历史状态：已完成（2026-07-11）；当时累计 64/64 自动检查通过。
- [x] 1900 起始时间、暂停、1/2/4/8 倍速、单步和周期事件。
- [x] 稳定事件队列和确定性时间测试。

## M2：核心数据和随机种子

- 目标：建立稳定实体模型、跨实体引用验证和统一确定性随机服务。
- 历史状态：已完成（2026-07-11）；当时累计 104/104 自动检查通过。
- [x] 稳定 ID、纯数据模型、两阶段数据验证和 RNG 状态恢复。

## M3：二维地图和动态控制区

- 目标：显示并更新双层地图、控制区和自动前线。
- 历史状态：已完成（2026-07-12）；当时累计 158/158 自动检查通过。
- [x] 2 国、8 地区、80 控制单元、城市、铁路和前线。
- [x] 增量前线、控制易手、孤立和包围测试。
- [x] P0-R1 模拟质量修复：铁路、地区社会影响、多方向进攻和包围进入控制压力倍率。
- [ ] 重新执行地图边界、压力和一年模拟回归，确认新倍率没有破坏原控制节奏。

## M4：随机人物和人物信息

- 目标：按国家和三种模式生成并展示人物。
- 历史状态：已完成（2026-07-12）；当时累计 201/201 自动检查通过。
- [x] 资质、技能、性格、倾向、状态和信息可见性。
- [x] P0-R1 模拟质量修复：全人口模式按国家总人口、人口群体人数和群体职业结构抽样；国家选择与人物生成使用隔离的确定性随机流。
- [ ] 本机验证 120 人分布、相同种子复现和背景人物层级往返。

## M5：行动和必然成功机制

- 目标：实现八类按时间推进、可准备、可中断并能达到必然成功的行动。
- 历史状态：底层服务和旧检查完成；普通玩家闭环仍由 P0-R1 验收。
- [x] 工作量、效率、暂停、取消、中断、结果和阈值。
- [x] 权威行动依赖重算、费用与创建原子化、领域失败奖励回滚。
- [x] P0-R1 模拟质量修复：玩家可明确选择额外财富投入，提高资金支持和准备度；投入写入行动审计。
- [ ] 在 1280×720 中验证投入控件、八类目标、费用、暂停和保存加载。

## M6：组织、职位、关系和简化 AI

- [x] P0-R1 模拟质量修复：NPC 保存并推进真实长期行动实例，承担同类费用、工作量和结果规则；可从空关系网络建立第一条关系；组织获得有上限的月度收入。
