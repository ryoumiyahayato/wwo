# 《1900》重设计实施路线图

状态：设计冻结 v1。阶段 0 完成后必须等待用户批准；在批准前不得进入 UI 空壳原型或正式业务实现。

## 1. 总规则

- 一次只实施一个阶段；发现后续需求只记录接口。
- 每阶段只增加一至两个核心因果循环。
- 先数据结构和服务边界，再静态 UI，再最小闭环，再扩内容。
- 不同时重写人物、市场、地图和战争。
- 正式 UI 不承载模拟公式，不通过隐藏按钮冒充玩家流程。
- 每阶段分别记录自动验证、实际窗口证据、性能和未验证范围。
- 任何阶段不得因现有代码难以适配而修改人物锚定、有限认知或世界自主原则。

## 2. 阶段依赖总览

```text
0 设计冻结
└─1 静态 UI 空壳
  └─2 住户、时间与生活状态
    └─3 有限认知、空间与关系卡
      └─4 就业、企业现金流与软失败
        └─5 组织权限、法律与地方制度
          └─6 区域市场
            └─7 真实世界地图
              └─8 国家议程、政治联盟与合法性
                └─9 战争升级、动员与陆军前线
                  └─10 技术形成、扩散与采用
                    └─11 社会继承与长期世界
```

区域市场先在稳定的抽象地区 ID 上建立资金与商品守恒；真实地图随后替换地理表现和空间数据，不反向改写经济语义。

### 首版重构可玩里程碑

阶段 1～5 构成首版必须实现的新版人物闭环：静态架构获批后，完成住户/时间/生活状态、有限认知/空间/关系、就业/企业现金流/软失败，以及组织权限和地方制度。此时农民、工人、贫困者和小商人应能在不进入国家高位的情况下持续游戏。

阶段 6～11 在首版闭环中先保留稳定接口，再按顺序实现区域市场、真实地图、国家政治、战争、技术和长期继承。保留接口不得以空按钮、假数据或无作用字段出现在正式玩家流程。

## 3. 阶段 0：设计冻结（本轮）

### 交付

- 17 份权威设计文档。
- 11 张 1280×720 低保真线框。
- 五层权限、时间、住户、信息、空间、法律、失败、AI、市场/战争/技术接口。
- 可复用与待替换系统清单。

### 禁止

不修改正式业务服务、存档格式、场景、市场、战争、真实地图或正式 UI。

### 验收

- 文档覆盖全部指定主题，术语与稳定 ID 规则一致。
- 旧原型与未来设计的事实边界明确。
- 内部链接和文件路径有效。
- 用户明确批准后才进入阶段 1。

## 4. 阶段 1：静态 UI 空壳原型

### 范围

四角布局、全屏地图占位、人物/国家/时间/动态角落组件、人物/组织/地区卡、11 类线框的静态交互、悬停和点击详情。

全部使用静态假数据，放入独立原型目录。不得连接正式社会模拟或修改正式存档。

### 验收

- 1280×720 无裁切，较大窗口正确锚定。
- 地图保持全屏背景；四角组件不组成整排管理栏。
- 左键是主要操作；Esc 分层关闭。
- 上下文动作不平铺；未知人物不出现。
- QA 与正式信息层在原型中有明确视觉边界。
- 用户批准空间和信息密度后才进入阶段 2。

## 5. 阶段 2：住户、时间与生活状态

### 核心循环

住户收入与支出；工作/通勤/照料/休息的 168 小时时间预算；健康、疲劳和压力；生活水平与住房。

### 迁移策略

在正式版本化模型中新增住户、住所、财务账户和时间承诺。旧单一财富/行动字段只通过明确迁移层读取，不作为新权威字段。

### 验收

- 每个存活人物恰有一个住户。
- 正常就业者不点击“工作”也获得来自雇主的基础工资。
- 住户支出有收款方，现金和债务守恒。
- 什么都不做产生基础恢复；超负荷有健康/表现后果。
- 一个主要计划、最多两个次要活动，不能超卖时间。
- 推荐开局 12 个月无无来源财富、无确定性生存死局。

## 6. 阶段 3：有限认知、空间与关系卡

### 核心循环

知识来源/可信度/过时；当前/常住/工作地点；通勤、基础旅行和通信；直接关系、可接触、公开与未知人物；人物卡上下文互动。

### 验收

- 其他人物不泄露精确财富、健康、技能和完整关系。
- 每条非本人事实显示来源和时间。
- 地图选择不改变个人行动目标。
- 同地、通信、引荐和旅行产生不同接触成本。
- 已有关系可从卡片继续互动，无全人物下拉列表。
- AI 不能选择自己不知道或无法到达的对象。

## 7. 阶段 4：就业、企业现金流与软失败

### 核心循环

正式就业合同、工作表现、企业工资/现金/债务、岗位与裁员；失业、欠薪、破产、住房风险及恢复路径。

首版只需单产品聚合企业和基础账户，不实现区域商品市场价格。

### 验收

- 工资从企业账户支付；现金不足产生欠薪/借款/裁员/破产。
- 企业破产改变员工、债权人与资产，不结束人物游戏。
- 失业和破产各有至少两条权威可达恢复路线。
- 推荐和困难开局都标注四类难度。
- 多种子 24 个月测试无确定性死局、资金守恒误差为零。

## 8. 阶段 5：组织权限、法律与地方制度

### 核心循环

我的/探索组织、具体职位、作用域权限、组织项目；地方机构、法律权利、预算/人员、行政/执法能力和地方落实。

### 验收

- 成员身份与职位分离，权限有来源、范围和有效期。
- 职位不能绕过预算、辖区和程序。
- 组织项目消耗人物时间与组织资金。
- 一项地方政策完整经过提出、批准、预算、执行和落实。
- 企业至少表现出遵守与规避两种响应。
- 被免职产生恢复玩法，不触发 Game Over。

## 9. 阶段 6：区域市场

### 核心循环

8 类商品、生产、库存、住户消费、跨区运输、价格、工资、失业和企业利润。

### 验收

- 每笔交易具有买方、卖方和资金流。
- 价格对供需、库存和运输变化有界响应。
- 铁路/港口/道路改变运输成本和容量。
- 住户生活篮子影响实际健康/压力，不要求逐件购买。
- 企业利润、工资和税收可从市场账本解释。
- 五年多种子市场无 NaN、负库存、无来源现金或单周期价格爆炸。

## 10. 阶段 7：真实世界地图

### 核心循环

以真实海岸线、1900 国家、150～250 宏观地区、城市、铁路和港口替换架空网格；接入空间、市场、人口与机构读取。

### 验收

- 来源、许可、版本和转换过程完整。
- 世界/国家地区/地方三尺度语义切换。
- 八类地图模式只读显示，不越权操作。
- 平移缩放、命中、标签和 LOD 达到地图性能预算。
- 活跃人物、背景人物与人口群体的升降级保持总量与稳定 ID。
- 旧架空国家不作为正式内容进入新游戏。

## 11. 阶段 8：国家议程、政治联盟与合法性

### 核心循环

利益群体、组织立场、联盟承诺、政体程序、国家议程、合法性和国家/地方权限。

### 验收

- 一项国家议程有支持者、反对者、财政、受益/受损和执行难度。
- 高级职位仍需联盟、程序和执行能力。
- 议程失败、联盟破裂和免职产生后续路线。
- 普通人物可观察并影响公开政治，但不能直接提交国家决定。
- 玩家不行动时 AI 联盟和议程仍连续推进。

## 12. 阶段 9：战争升级、动员与陆军前线

### 核心循环

利益冲突、外交危机、准备、动员、宣战/妥协、陆军战区、前线、补给、控制、和谈和战后状态。

### 验收

- 战争不会无前兆随机出现。
- 和平无前线纹理与军事控制压力。
- 战争状态含参战方、开始时间、目标、战区、和平条件与僵持原因。
- 作战读取兵力、装备、训练、指挥、补给、地形、士气与准备。
- 平民、企业、士兵、军官和政府职位具有不同体验与权限。
- 战败产生占领、迁移、失业和政治路线，不自动 Game Over。

海军与空军只保留数据和权限接口，不在本阶段扩展完整玩法。

## 13. 阶段 10：技术形成、扩散与采用

### 核心循环

知识形成、贸易/教育/人员传播、组织采用、机构定向研究和技术引进。

### 验收

- 前置符合因果关系，不靠一次随机事件永久解锁或锁死。
- 被动扩散在玩家不研究时仍能发生。
- 定向研究需要机构、预算、人才和权限，投入边际递减。
- 掌握知识与实际采用分开。
- 技术改变企业、基础设施、通信或军事的实际参数，并有采用成本。

## 14. 阶段 11：社会继承与长期世界

### 核心循环

退休、死亡、长期失能、接班准备、合法社会继承者、多年住户/组织/世界连续性。

### 验收

- 年轻健康人物不显示常驻传承入口。
- 候选只来自亲属、门生、长期伙伴、合伙人、组织继任者或政治盟友。
- 控制权、财产、住户身份和组织权限分别处理。
- 永久退出且无合法候选才 Game Over。
- 多代存档、人物升降级、目标与知识历史保持一致和有界。

## 15. 跨阶段质量门

每个阶段提交前：

- `git diff --check`。
- 受影响自动测试与统一验证。
- 日志解析/类型/脚本加载扫描。
- 1280×720 可见控件旅程。
- 对应时间范围的模拟和资金/状态不变量。
- 若修改存档，真实旧档迁移、失败回滚和重新保存。
- 若修改模拟公式，重新建立性能基准。
- 未运行、未人工验证和未导出的项目明确写“未运行/未人工验收/未完成 Release 冒烟”。

## 17. 阶段 0 完成语句

本轮正确状态只能是：

> 游戏整体设计与信息架构已经冻结，等待用户批准后进入 UI 空壳原型阶段。

补充中间层的等价状态为：

> 人物生活、时间、信息、空间、制度、失败恢复和 AI 中间层设计已经冻结，可以与此前总设计共同进入 UI 空壳原型阶段。
