# 世界城市细化数据规则

## 目标

正式地图在三个层面使用不同密度的数据：

- 世界层只显示首都、超大城市和现有人工策划节点；
- 区域交通层显示欧洲、亚洲、北美人口不少于 5,000 的城市或行政中心；
- 城市层允许显示当前视窗内的全部已加载节点；法国额外提供全国市镇级记录。

城市细化不得把数万条记录一次性并入 `data/world_map/cities.json`。该文件继续只保存人工策划、历史主题和中文名称覆盖。大规模数据放在 `data/world_map/city_detail/`，由索引和分片组成，运行时只加载当前视窗相交的分片。

## 数据来源与许可

生成器使用 GeoNames 官方下载文件：

- `cities5000.zip`：欧洲、亚洲、北美的一般城市层；
- `FR.zip`：法国 `ADM4` 市镇记录，以及人口较大的聚居地和行政中心；
- `countryInfo.txt`：国家与洲别筛选。

生成结果必须保留 GeoNames 和 CC BY 4.0 署名。索引与每个分片都写入来源、许可和生成时间。

## 历史边界

这批坐标、人口和行政编码是现代参考数据，不是 1900 年历史复原。所有记录必须包含：

```json
{"modern_geography":true}
```

总索引必须包含：

```json
{"historical_status":"modern_reference_only"}
```

现有人工城市表可以覆盖中文名、历史主题名和显示优先级，但不得因此把整条现代记录标记为历史事实。历史行政边界、当时城市名称和人口需要以后由独立历史数据覆盖层处理。

## 法国市级标准

法国分片必须至少包含 25,000 条 `ADM4` 市镇记录；生成数量低于这一安全下限时构建失败。法国按 `admin1_code` 分片，避免进入法国城市层时一次解析全国全部市镇。

一般城市记录与市镇同名时优先保留市镇记录；首都和行政中心仍可作为单独高优先级城市节点存在。

## 运行时预算

默认预算写入索引：

- 同时缓存的分片国家/区域：12；
- 单帧可见细化节点：1,600；
- 单帧细化标签：180。

超过预算时按以下顺序保留：

1. 当前选中或玩家所在城市；
2. 人工策划名称覆盖；
3. 首都和行政中心；
4. 人口和标签优先级；
5. 稳定 ID。

标签必须经过屏幕矩形碰撞检查，不能为显示全部城市而互相覆盖。

## 更新流程

`tools/world_map/generate_city_shards.py`只接受本地下载文件并产生确定性结果。GitHub Actions负责下载官方源、运行自检、生成数据、验证数量与字段，再把结果提交到当前地图开发分支。

任何数据更新都必须同时通过：

- 生成器自检；
- 输出结构验证；
- 法国市镇数量下限；
- 地图冷加载和热缓存性能测试；
- Windows导出测试。
