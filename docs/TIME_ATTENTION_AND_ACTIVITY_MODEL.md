# 时间、注意力与活动模型

状态：设计冻结 v1。保留现有整数权威小时作为世界时间基础，但正式人物行为不再全部归入单一“长期行动”。

## 1. 两种时间

- 世界时间：自 1900-01-01 起的整数权威小时，决定日历、周期和事件顺序。
- 人物时间预算：人物每周可用于义务、休息、计划和互动的有限小时。

世界时间推进不代表人物可以无限并行行动。人物时间只在明确边界分配，不要求玩家逐小时排班。

## 2. 每周时间预算

每个可行动人物每周总计 168 小时，按下列桶结算：

```text
工作义务
+ 通勤
+ 家庭与照料义务
+ 基础生活事务
+ 基础休息与睡眠
+ 主要计划承诺
+ 次要活动承诺
+ 未分配恢复时间
= 168 小时
```

各桶使用真实小时以便解释，但 UI 默认显示为“占用很低/适中/很高”和剩余可支配时间。具体默认睡眠、生活事务和职业工时由数据配置决定。

若承诺超过可用时间，系统不静默超卖。玩家必须减少计划、降低强度、购买服务、请假或接受明确的休息不足与违约后果。

## 3. 六类活动

### 日常义务

正常上班、通勤、基础生活、家庭照料、债务偿还和普通休息自动持续。玩家只在改变合同、住户安排或强度时操作，不需要每天点击。

### 个人计划

学习、求职、休整、搬迁、考试准备、写作、创办企业和个人投资。首版同一人物最多一个主要计划。

### 人物互动

联系、拜访、帮助、引荐、谈判、调查和合作，持续数小时至数日。首版最多同时承诺两个次要活动；重复互动受通信、旅行和关系冷却限制。

### 组织项目

招募、宣传、筹款、活动、内部改革、扩建和行业项目。项目占用人物时间，也占用组织预算和资源。

### 制度决策

提案、审议、预算、任命、动员、外交和军事命令。作出决定本身可能很短，但准备、联盟和执行是独立计划或机构项目。

### 反应事件

疾病、解雇、企业倒闭、战争、关系冲突、拘留和继承机会。事件可以抢占计划、触发自动暂停或要求在期限内答复。


已经过去的区间使用该区间开始时保存的效率、权限和上下文结算。新状态只从明确边界生效，延续现有行动系统已经验证的时间语义。

## 5. 强度与取舍

活动强度同时改变进度、疲劳、压力、失败后果和其他义务表现：

- 学习更多：成长更快，但减少恢复时间。
- 兼职：提高收入，但增加疲劳和通勤。
- 高职位：扩大权限，也增加固定职责与政治压力。
- 家庭照料：减少可支配时间，但履行关系和住户义务。
- 富裕：可购买交通与照料服务释放时间，但必须支付真实主体。

进度不是简单的“投入小时 × 技能”。结构为：

```text
有效进度 = 承诺小时 × 强度效率 × 能力修正 × 状态修正
           × 资源/环境修正 × 连续性修正
```

所有系数数据驱动，并记录主要来源供因果解释。

## 6. 疲劳与恢复连接

每周负荷由工作、通勤、照料、旅行、计划、战斗和休息不足构成。每日结算短期疲劳，周边界重新核对时间预算。

普通未分配时间自动形成恢复，不要求点击“休息”。主动“休整”是牺牲其他计划以加快恢复、接受医疗或改善生活状态。

连续超负荷产生：

- 工作或计划表现下降。
- 压力增加。
- 缺勤和关系义务失败。
- 长期健康损害。

## 7. 抢占、暂停与恢复

事件按照严重度决定：

- 提醒但不抢占。
- 暂停次要活动。
- 暂停主要计划。
- 中断工作或旅行。
- 使人物暂时不可行动。

暂停保留已完成进度和已承诺但未消耗资源；取消根据合同和活动规则产生退款、损失或声誉后果。恢复时重验地点、权限、目标和资源。

## 8. 旅行与通信占用

当面活动包括往返旅行小时；信件、电报和电话消耗较少人物时间，但有费用、基础设施、信息容量和回复延迟。跨地区活动不能只增加一个抽象阻力而不记录时间与空间来源。

## 9. AI 使用

AI 先为日常义务和最低恢复预留时间，再将剩余时间投入与当前长期目标一致的中期计划。短期事件可以抢占，但事件结束后应恢复原计划或明确放弃。

AI 不每日重抽全部活动。每个承诺记录已投入时间、成本、进度和放弃条件；除非条件触发，不频繁切换无关目标。

## 10. UI

人物中心显示：本周义务、主要计划、次要活动、可支配时间和过载警告。第一层不显示 168 小时明细；点击后显示每项来源。

开始新活动时预览：每周时间、预计历时、对休息/工作/家庭的主要影响。超出预算时必须先选择调整，不允许按钮提交后才发现没有时间。

## 11. 结算周期

| 周期 | 内容 |
| --- | --- |
| 小时 | 旅行、战斗、临近期限和必要的连续进度 |
| 每日 | 疲劳、恢复、短期健康、主要计划累计 |
| 每周 | 时间预算、工作表现、关系维护、活动承诺重验 |
| 每月 | 合同、财务、组织项目、长期计划评估 |
| 事件边界 | 权限、地点、目标、资源与中断状态切换 |

不允许在 `_process()` 或 `_physics_process()` 中遍历全部人物结算时间预算。

## 12. 首版边界

首版必须：日常义务自动化、168 小时预算、一个主要计划、最多两个次要活动、恢复时间、过载后果、事件抢占、AI 承诺连续性。

只保留接口：多人协同日程、精确班次、轮班表、跨时区、复杂军事值勤和逐日历预约。首版不提供逐小时排班 UI。
