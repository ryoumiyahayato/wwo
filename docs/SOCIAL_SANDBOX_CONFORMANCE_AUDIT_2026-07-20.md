# 社会沙盒运行逻辑符合性审计（2026-07-20）

## 结论

当前正式入口已经形成一条可以实际运行的社会行动闭环，但还不能据此宣称《1900》社会沙盒核心重构正式开发指令中的全部要求已经完成。

当前准确状态是：

- 正式产品入口、已有权威服务复用、处境推导、稳定目标、玩家显式计划、NPC计划、日程与旅行、任务结算、重要事件、有限认知、延迟反应、存档和长时间运行已经接通。
- 当前实现能够在玩家不操作时产生社会任务与事件，并保持确定性和历史上限。
- 当前实现仍是有限领域规则驱动的里尔社会沙盒，不是所有现实行动、所有阻碍、所有证据关系和所有组织生命周期都已经实现。
- 同一小时任务已经统一准备并解决资源冲突，但当前不是完整的多阶段即时反应内核。
- 每项行动提交具有自己的权威状态快照和回滚；当前不是“同一小时全部无关行动共享一个全有或全无事务”。这是为了防止一项失败撤销其他人物已经成立的无关事实。
- 重要事件只有单一 `cause_event_id`；尚未实现多个直接原因组成的因果网络字段。
- 自动测试覆盖了一部分关键性质，但没有逐条证明正式指令列出的 80 项要求。

因此，本轮将原先会产生误导的旧社会沙盒测试替换为正式 V2 组合根符合性测试，并把未满足部分明确记录下来。

## 删除与替换说明

本轮没有删除正式运行代码。

`tests/v2_3/v2_3_social_sandbox_test.gd` 的原内容被完整替换，原因如下：

1. 原测试创建 `V23ProductSimulation`，而普通产品入口使用 `V23ProductSimulationV2`。
2. 原测试要求准备程度 1000 产生跨种子必胜，与正式 V2 已取消必胜捷径相矛盾。
3. 原测试把 `factory_fives_workgroup_three:delegate` 当作预设主要结果，并要求儒勒或路易取得该职位。
4. 原测试通过 `force_set_at_location` 把人物直接搬到目标地点后证明职位和行贿结算，不能证明正式旅行与日程链成立。
5. 正式替代路径是新的同名 V2 符合性测试，加上 `v2_3_social_sandbox_completion_test.gd`。
6. 替换只影响测试断言，不改变存档 Schema、正式 ID 或用户存档。

| 要求 | 当前情况 | 缺口 |

| 所有现实成立的方法都可进入统一管线 | 当前目录约 32 种方法 | 尚未覆盖正式文件读写、完整借贷协商、介绍链、组织命令、暴力结果等全部类别 |

## 一个当前确实成立的因果链

```text
人物从住户现金、劳动合同、组织职位和已有认知推导处境
→ 处境形成稳定目标
→ 玩家或 NPC 选择适用方法
→ submit_intent 创建意图
→ 为行动者和对象寻找共同时间
→ 不在当地时通过正式旅行服务建立行程
→ 双方日程预留社会行动
→ 到期重新检查人物到场、日程、职位与资源
→ 同小时提案按得分和稳定任务 ID 排序
→ 人物时段和唯一职位冲突统一解决
→ 单项状态改变与事件提交失败时恢复权威快照
→ 成功或失败事实写入只追加事件账本
→ 目击和消息转化为人物知识
→ 相关人物被标记为需要重新评估
→ 延迟反应重新形成目标和任务
```

该链实际涉及权威时间、具体地点、旅行路线、双方日程、人物关系、人物知识、组织成员与职位、现金账本、任务和事件顺序。

## 当前不能使用的完成表述

不得再写：

- “全部 80 项验收已经通过”；
- “所有现实行为都已经支持”；
- “完整多阶段因果内核已经完成”；
- “人物已经具备完整错误认知和欺骗推理”；
- “地图、计划和人工验收材料已经完成”；
- “完整 Alpha 功能与性能验收通过”。

当前可以写：

> 正式 V2 社会沙盒的基本因果闭环已经接入现有权威服务并通过当前自动回归；完整开发指令中的方法广度、多阶段反应、因果网络、认知复杂度、地图可玩性和人工验收材料仍需继续实现和验证。
