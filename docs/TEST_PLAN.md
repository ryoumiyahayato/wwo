# 测试计划

## 原则

使用 Godot 自包含测试运行器，不依赖第三方插件，不只依靠手动点击。失败测试返回非零退出码，确定性系统使用固定输入。

- Compatibility 渲染设置和主场景路径。
- 全部核心 Markdown 文档存在。
- 日志等级过滤和格式。
- 主菜单场景可加载、实例化且包含退出按钮。

2026-07-11 结果：25/25 通过；headless 主场景和 Compatibility 窗口短时启动均以退出码 0 结束，未观察到解析错误或阻塞警告。另以 Movie Maker 渲染并人工检查 1280×720 首帧，中文标题、四个菜单入口、禁用态、退出按钮和安全边距显示正常。

## M6 自动检查

- 两国共 8 个组织及政府、军队、企业、工会四类；三级职位、槽位、组织关系与权限。
- 技能不能替代职位权限；加入/离开、入口职位、晋升和唯一领导槽位。
- 120 名世界背景人物、玩家与 8 名活跃领导、20 人上限和稳定 ID 升降级。
- 背景轻量记录不含完整 AI；升级补全活跃字段，降级移除 AI，玩家和领导不可降级。
- 人物对关系按需创建、反向复用、增量更新、数值范围和人物关系 ID。
- 活跃 NPC 每日决策、实际月边界长期目标、有限候选确定性、高疲劳恢复选择和调试状态。
- 社会面板默认隐藏 AI 调试，显式开关显示目标、候选权重、最终选择和下次决策。

## M7 自动检查

- 组织社会/军事支援分别检查职位权限、消耗资源并只进入对应影响通道。

- 五类退出原因和不同财富、声望、情报、盟友/敌对关系及职位继承比例。
- 候选只来自真实关系和共同组织；弱联系、外国、拘押和已退出人物被排除；排序确定。
- 继承保持人物稳定 ID、总人物数、地图与社会服务对象，不复制技能或完整人物数据。
- 前人物进入历史索引，未完成行动清除，继承者成为玩家后移除 AI。
- 社会面板候选、退出原因、确认继承和已退出计数。
- 100 条关系的候选生成和部分继承性能基线。

## M9 自动检查与运行验收

- 行动面板按类别提供人物、组织或控制单元目标；建立关系、加入组织、争取职位和地区政策结果实际写回权威服务。
- 自动闭环覆盖关系、组织、职位、地区影响、退出、关系候选继承、保存、加载和继续推进。
- 8 倍速度先推进 30 日，再完成 8760 小时一年模拟；保持 20 人活跃上限、19 个 NPC AI 和 100 条稀疏关系。
- 一年触发 365 次每日 AI、12 次月度 AI 和 52 次单自动档；结算日志有界，已记录单次结算均低于 100 毫秒。
- 默认 1280×720 配置、Windows x86-64、Linux x86-64 和 macOS Universal 预设结构。
- Headless 主场景及 1920×1080 Compatibility 窗口在 Windows 11 开发机退出码均为 0。

## P0-R1 状态一致性回归

覆盖：

- 背景人物升级、成长、降级、再次升级后，技能、隐藏资质和真实倾向保持一致。
- 入口职位满员时加入组织失败，成员列表、人物组织索引和职位索引不产生半完成状态。
- 正式玩家行动费用扣除与行动创建为单一事务，创建失败恢复财富。
- 进行中行动的目标资格、职位权限、关系、组织支持和目标阻力在权威小时边界重新验证。
- 领域效果无法写入时，撤销通用成功奖励并应用失败结果。
- 进行中行动存档中的权限、支持、准备、阻力和费用重新对照权威世界验证。
- 继承失败恢复人物、组织、关系和 AI；活跃层已满时仍可先释放旧玩家槽位完成继承。
