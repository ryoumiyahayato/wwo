# 测试计划

## 原则

使用 Godot 自包含 `SceneTree` 测试脚本，不依赖第三方插件。失败必须返回非零退出码；确定性系统使用固定输入。历史通过记录只说明当时提交，不自动适用于当前默认分支。

Godot Headless 进程返回 `0` 不是充分条件；日志中出现脚本解析、类型解析或脚本加载错误，同样判定失败。干净检出必须先运行编辑器导入以生成全局 `class_name` 缓存。

## 固定本机环境

- Godot：`4.6.3.stable.official.7d41c59c4`
- 渲染器：Compatibility
- 项目：`D:\wwo`
- 默认视口：1280×720

## 统一自动验收入口

```powershell
powershell -ExecutionPolicy Bypass -File 'D:\wwo\tools\run_validation.ps1'
```

脚本按以下顺序执行：

1. Headless 编辑器导入和全局类扫描。
2. 原有综合回归 `tests/test_runner.gd`。
3. P0-R1 逻辑回归。
4. P0-R1 玩家旅程。
5. P0-R1 安全回归。
6. 状态一致性回归。
7. 模拟质量回归。
8. Codex 审计专项回归。
9. Headless 主项目启动和解析日志检查。

## 各测试覆盖

### 原有综合回归

覆盖原 M0 至 M9 的项目配置、时钟、数据、地图、人物、行动、组织、AI、继承、存档、开发工具和长期模拟。历史 `564/564` 只能作为旧提交参考。2026-07-13 Codex 影子验证在只修复编译阻断后为 `552/564`；其中多项是旧夹具与新权威规则不同步，必须在当前默认分支重新运行后逐项判断。

### P0-R1 逻辑回归

覆盖正式行动入口、小时推进、自动存档、中断顺序、调查、职位空缺、前线规则和存档篡改。涉及正式进行中行动的测试必须通过原子费用入口创建。

### 真实玩家旅程

覆盖普通玩家从正式入口完成：建立关系、加入组织、争取职位、地区行动、合法退出、继承、保存退出和重新加载。开始按钮已经固定在滚动区外，但仍需在 1280×720 实际窗口复核。

### 安全回归

覆盖事务恢复、配置版本、非法行动、组织/关系双向索引、领域写回一次性和备份恢复。备份快照按 JSON 解析后的语义比较，避免把整数/浮点归一化差异误判为状态漂移。

### 状态一致性回归

覆盖：

- 背景人物技能、资质和倾向跨升降级保持连续。
- 组织加入与入口职位分配原子化。
- 行动创建失败回滚费用。
- 进行中行动重新验证目标、权限、支持和阻力。
- 领域写入失败撤销成功奖励。
- 行动存档对照权威世界验证。
- 继承失败回滚和满活跃层继承。

### 模拟质量回归

覆盖人口结构、主动投入、NPC 长期行动、关系自举、组织经济、生命周期、地区辖区、控制倍率、继承合并和跨页面时钟。

### Codex 审计专项回归

```powershell
& 'D:\Tools\Godot-4.6.3\Godot_v4.6.3-stable_win64.exe' --headless --path 'D:\wwo' --script res://tests/codex_audit_regression.gd
```

覆盖：

- 战略地图和强类型地图依赖可解析实例化。
- 开始行动按钮不位于滚动区内。
- 学习行动可以选择政治、行政、军事等任意人物技能。
- 非学习行动产生对应主技能的实践成长。
- 普通模式与完整人口模式共 1000 种种子，在技能训练、准备和资金充分后，全部核心行动达到保证成功线。
- 年轻健康人物不能伪造死亡、退休、长期监禁或严重失势。
- NPC 已过去时间使用旧效率结算，新条件只从日边界生效。
- 超过活跃上限、激活种子缺失、AI 状态缺失或行动实例 ID 重复的存档被拒绝。

## 人工交互验收

在 1280×720 Compatibility 窗口中完成：

1. 新游戏创建普通模式和完整人口模式人物。
2. 打开长期行动，选择至少三种不同技能学习，确认完成后只增长所选技能。
3. 通过学习和实际行动提高 `political_activity`、`administration` 和 `military_command`。
4. 在精通技能、满准备和充分资金下，确认职位、政策和控制行动显示“必然成功”。
5. 验证无职位时政策行动不可开始；获得合法职位后只能选择辖区内目标。
6. 验证年轻健康人物的退出原因列表只包含自愿退出；达到年龄、拘禁或失势条件后相应原因出现。
7. 推进至少 30 游戏日，观察 NPC 行动进度、关系、组织资源和地区变化。
8. 打开人物信息页面，在未暂停状态停留，确认日期继续推进。
9. 保存、完全退出、重新启动并加载，确认玩家与 NPC 行动、组织、关系、生命周期和地图状态一致。

## 长期与性能验收

### 30 日

- 活跃人物不超过 20，且每个活跃 NPC 都具有 AI 状态。
- NPC 长期行动可开始、推进、完成和失败；不存在每日瞬时免费奖励或新状态追溯旧时间。
- 至少产生一条新 AI 关系或一个真实组织/职位变化。
- 组织资源不会全部归零，也不会超过配置上限。
- 单次日/月结算不超过 100 毫秒。

### 一年

- 推进 8760 小时，记录日、月、周和自动存档次数。
- 人物年龄增加 1，符合条件者退休或死亡，领导空位能补充。
- 活跃人物、AI 覆盖、关系数量、行动 ID、存档体积和结算日志保持有界。
- 铁路、社会支持和包围倍率不会造成前线无条件快速崩溃。
- 组织资源围绕收入与支出变化，不无限增长或永久停滞。
- 存档往返后继续推进得到相同权威结果。

## 导出验收

全部自动、人工和长期验收通过后执行：

```powershell
& 'D:\Tools\Godot-4.6.3\Godot_v4.6.3-stable_win64.exe' --headless --path 'D:\wwo' --export-release 'Windows Desktop' 'D:\wwo\builds\windows\wwo-p0-r1.exe'
```

在 Windows 11 独立启动导出 EXE，重复主菜单、新游戏、技能成长、保证成功、页面切换、30 日推进、保存加载和正常退出。
