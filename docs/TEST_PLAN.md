# 测试计划

## 原则

使用 Godot 自包含 `SceneTree` 测试脚本，不依赖第三方插件。失败必须返回非零退出码；确定性系统使用固定输入。历史通过记录只说明当时提交，不自动适用于当前默认分支。

Godot Headless 进程返回 `0` 不是充分条件；日志中出现脚本解析、类型解析或脚本加载错误，同样判定失败。干净检出必须先运行编辑器导入以生成全局 `class_name` 缓存。

## 固定本机环境

- Godot：`4.6.3.stable.official.7d41c59c4`
- 渲染器：Compatibility
- 项目：`D:\wwo`
- 默认视口：1280×720

## 统一自动验收入口

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File 'D:\wwo\tools\run_validation.ps1'
```

脚本按以下顺序执行：

1. 校验 Godot 精确版本。
2. Headless 编辑器导入和全局类扫描。
3. 当前综合回归 `tests/current_test_runner.gd`，继承完整 `tests/test_runner.gd` 套件。
4. P0-R1 逻辑回归。
5. 审计后 1280×720 玩家旅程。
6. P0-R1 安全回归。
7. 状态一致性回归。
8. 模拟质量回归。
9. Codex 审计专项回归。
10. 1000 种标准随机种子的正常预算前期可达性回归。
11. Headless 主项目启动和解析日志检查。

## 各测试覆盖

### 原有综合回归

覆盖原 M0 至 M9 的项目配置、时钟、数据、地图、人物、行动、组织、AI、继承、存档、开发工具和长期模拟。历史 `564/564` 只作为旧提交参考；当前夹具已迁移到权威领域写回、合法继承原因和真实行动存档入口，2026-07-15 为 `570/570`。

### P0-R1 逻辑回归

覆盖正式行动入口、小时推进、自动存档、中断顺序、调查、职位空缺、前线规则和存档篡改。涉及正式进行中行动的测试必须通过原子费用入口创建。

### 真实玩家旅程

覆盖普通玩家从正式入口完成：学习与实践成长、暂停/继续/取消、建立关系、加入组织、争取职位、地区行动、合法退出、继承、保存和重新加载。自动旅程固定 1280×720、关闭开发者模式，并验证八类行动、技能、投入和固定开始按钮真实位于首屏；实际窗口人工复核仍单独保留。

### 安全回归

覆盖事务恢复、配置版本、非法行动、组织/关系双向索引、领域写回一次性和备份恢复。备份快照按 JSON 解析后的语义比较，避免把整数/浮点归一化差异误判为状态漂移。

### 状态一致性回归

覆盖：

- 背景人物技能、资质和倾向跨升降级保持连续。
- 组织加入与入口职位分配原子化。
- 行动创建失败回滚费用。
- 进行中行动重新验证目标、权限、支持和阻力。
- 领域写入失败撤销成功奖励。
- 行动存档对照权威世界验证。
- 继承失败回滚和满活跃层继承。

### 模拟质量回归

覆盖人口结构、主动投入、NPC 长期行动、关系自举、组织经济、生命周期、地区辖区、控制倍率、继承合并和跨页面时钟。

### Codex 审计专项回归

```powershell
& 'D:\Tools\Godot-4.6.3\Godot_v4.6.3-stable_win64.exe' --headless --path 'D:\wwo' --script res://tests/codex_audit_regression.gd
```

覆盖：

- 战略地图和强类型地图依赖可解析实例化。
- 开始行动按钮不位于滚动区内。
- 学习行动可以选择政治、行政、军事等任意人物技能。

- 年轻健康人物不能伪造死亡、退休、长期监禁或严重失势。
- NPC 已过去时间使用旧效率结算，新条件只从日边界生效。
- 超过活跃上限、激活种子缺失、AI 状态缺失或行动实例 ID 重复的存档被拒绝。

### 标准开局预算回归

`tests/early_game_reachability_regression.gd` 对 1000 个标准随机种子使用正常初始财富、正式行动入口和真实工作收入运行。不得使用开发者工具、技能 100、无限财富或手工写入关系/组织身份。每个种子最多学习两次，并分别验证：30 日内工作或学习、60 日内第一条关系、90 日内第一个组织、180 日内职位有效路径与成功线。输出各阶段比例、失败原因、最差种子、平均花费、平均游戏时间和确定性死局数。

## 人工交互验收

使用全新 `user://`，关闭开发者模式，在真实可见、未遮挡、可获得焦点的 1280×720 Compatibility 控件中完成；不得直调服务或隐藏控件：

1. 创建标准新游戏。
2. 通过正式行动抽屉从事一次本职工作。
3. 通过正式人物目标建立第一条关系。
4. 通过正式组织目标加入第一个组织。
5. 确认行动完成后当前槽立即释放，摘要进入可关闭的最近结果，且不阻止配置下一行动。
6. 验证人物、行动、社会和世界动态均从左侧动画打开、互斥、当前入口可收起且 Esc 可关闭。
7. 推进三个月，查看右下角通知和世界动态历史，并确认没有内部字段。
8. 确认和平状态与国境/控制边界文字明确，支持控制行动显示“当前无战争或没有合法前线目标”。
9. 保存、完全退出、重新启动并加载，确认行动结果、关系、组织、世界动态和和平状态一致。

在第 1～7 项重新通过前，不继续人工验收职位、政策、控制、继承或破坏性存档流程。

## 长期与性能验收

### 30 日

- 活跃人物不超过 20，且每个活跃 NPC 都具有 AI 状态。
- NPC 长期行动可开始、推进、完成和失败；不存在每日瞬时免费奖励或新状态追溯旧时间。
- 至少产生一条新 AI 关系或一个真实组织/职位变化。
- 组织资源不会全部归零，也不会超过配置上限。
- 单次日/月结算不超过 100 毫秒。

### 一年

- 推进 8760 小时，记录日、月、周和自动存档次数。
- 人物年龄增加 1，符合条件者退休或死亡，领导空位能补充。
- 活跃人物、AI 覆盖、关系数量、行动 ID、存档体积和结算日志保持有界。
- 和平状态不产生军事控制变化；显式战争状态才允许铁路、社会支持和包围倍率参与增量控制，且不得无条件快速崩溃。
- 组织资源围绕收入与支出变化，不无限增长或永久停滞。
- 存档往返后继续推进得到相同权威结果。
