# 事件与 AI 世界模拟

状态：设计冻结 v1。本文替代“每天从有限候选随机选行动”的长期目标设计；现有确定性调度、分层人物和行动边界可以复用。

## 1. 世界自主性

玩家不是世界运行条件。即使玩家不工作、不学习、不加入组织或一生只是普通农民，住户、企业、组织、机构、国家、市场、外交和技术仍按自己的目标、承诺和周期变化。

暂停只停止权威时间；观察界面、玩家理解程度或当前打开页面不得决定系统是否结算。

## 2. 目标—承诺—计划

### 长期目标

持续数月或数年，例如维持家庭、清偿债务、稳定就业、扩大企业、获得组织职位、推动政治方向或确保战争安全。

### 中期计划

持续数周或数月，例如学习、求职、积累现金、建立关系、加入组织、完成引荐、推进项目。

### 短期反应

处理疾病、解雇、价格暴涨、战争、冲突、期限和突发机会。反应可抢占计划，但不会无理由抹去长期目标。

## 3. 选择逻辑

AI 先处理硬危机和义务，再从与长期目标一致的可达计划中选择。评分结构：

```text
计划价值 = 目标优先级 × 预期收益 × 可达性 × 信息可信度
           - 时间成本 - 资金成本 - 风险 - 义务冲突
           + 已投入连续性 + 关系/组织支持
```

同分使用稳定 ID 或确定性随机流。AI 不读取超出自身知识范围的真值，也不每天随机重新选择无关行为。

## 4. 人物目标

人物目标池包括生存、稳定收入、家庭安全、健康、职业、财富、关系、组织地位、政治/意识形态、战争安全和迁移。人格、阶层、职业、住户、关系、组织、知识和世界事件影响优先级。

低阶 AI 拥有与玩家相同的收入、时间、住房和恢复约束，不能获得每日免费效果。

## 5. 组织与企业目标

- 企业：现金流、盈利、偿债、生产、扩张、市场、招募、政策。
- 工会：成员、工资、工时、安全、立法、地区影响。
- 教育科研：人才、教学、研究、扩散和资金。
- 军队：战备、人员、装备、补给、任务与战争目标。
- 政府机构：法律义务、税收、服务、稳定、政策执行。

组织目标产生真实预算、人员和项目承诺，不直接修改结果字段。

## 6. 国家目标

国家根据制度、资源和局势追求稳定、财政、经济、秩序、安全、国际利益、技术、合法性和战争目标。国家 AI 通过机构议程、联盟、预算和执行计划行动，而不是作为单一万能代理。

## 7. 多精度 AI

| 层级 | 状态与决策 |
| --- | --- |
| 活跃命名人物 | 完整目标、知识、时间预算、计划和反应 |
| 背景命名人物 | 身份、职业、组织、关系摘要与低频长期计划 |
| 人口群体 | 收入、需求、就业、迁移、健康、支持的聚合流 |
| 关键组织/机构 | 完整账户、目标、项目和权限 |
| 普通组织 | 月/季度聚合目标和项目摘要 |

升降级保持稳定 ID、长期目标摘要、已投入资源、关系与关键经历。不得激活后变成无历史新人物。

事件按状态触发，时间负责检查节奏。模板通过人物、地区、组织、市场和结果参数生成不同实例，不手写 2000 个近似文件。

长期内容目标可达到 1000～2000 个可区分事件结果，但这是模板、角色、地区、组织、选项和后果参数的组合规模。第一批实现只覆盖当前阶段的核心循环，并为每个模板设置冷却、重复限制和可测结果；不得用大量近似文本冒充系统深度。

## 9. 玩家与 AI 事件

- 玩家重要事件显示选项、期限、后果范围和已知信息。
- 普通 AI 大多数事件按目标、性格、知识和条件结算，不弹玩家窗口。
- 重要 AI 的国家、企业、军队或组织事件可进入世界新闻。
- AI 决策使用与玩家相同的领域规则、资金流和权限检查。

## 10. 事件层级

个人、住户/关系、工作、企业、组织、地区、市场、国家、国际、战争和技术事件各有类别。事件类别决定检查周期、可见性、聚合和历史保留，不决定其结果必然公开。

## 11. 冷却与重复

每个事件模板定义主体冷却、对象对冷却、地区冷却和最大重复。重复事件优先聚合为状态趋势；只有条件或后果发生实质变化时再次打断玩家。

同一人物不能每周重复遭遇完全相同的“偶然机会”而无限获利。

## 12. 时间周期

| 周期 | 主要用途 |
| --- | --- |
| 小时 | 旅行、临近期限、短期战斗和必要连续进度 |
| 每日 | 疲劳、恢复、短期健康、活动推进、危机反应 |
| 每周 | 时间预算、工作表现、关系维护、新闻、小型组织行为 |
| 每月 | 工资、生活费、市场、企业、组织账户、就业、迁移 |
| 每季度 | 国家议程、企业投资、技术扩散、大型项目、外交重评 |
| 每年 | 年龄、长期健康、人口、制度、退休与继承候选 |

继续采用明确权威顺序与事件队列；新增系统必须注册周期，不在自身 `_process()` 推断日期。

## 13. 世界消息聚合

结算记录先进入因果层，再按重要度和公开范围生成通知、事件或新闻。同类普通组织、就业、市场和迁移变化按地区与周期聚合。

世界动态不能成为开发日志。它只显示玩家通过媒体、组织或亲历获得的公开或相关信息。

## 14. 确定性与随机

相同初始状态、输入、随机种子和事件顺序必须复现。随机只能通过统一服务并按系统/实体隔离流，避免某个 UI 查询改变世界随机序列。

随机影响机会与结果分布，但关键生存、技术和恢复路线不得由一次坏运永久锁死。

## 15. 性能边界

- 不在 `_process()` 或 `_physics_process()` 遍历全社会。
- 背景人物不运行每日完整 AI。
- 月/季度批处理分块执行并有主线程时间预算。
- 候选先按知识、空间、权限和目标过滤，再评分。
- 关系保持稀疏；事件索引按状态标签与周期触发，不全表扫描。
- 正式运行不输出逐小时日志。

## 16. 可验证性

测试必须覆盖计划连续性、危机抢占后恢复、信息限制、资金守恒、权限重验、玩家不行动时世界变化、多精度升降级和跨存档确定性。

长时测试同时输出：目标完成/放弃原因、死锁、事件重复率、财富守恒误差、活跃实体数、结算耗时和消息聚合率，而不是单一通过数。

## 17. 首版与保留接口

首版必须：人物/住户/企业/组织的目标—承诺—计划，危机反应，多精度人物，模板事件，冷却，聚合消息和玩家不行动回归。

只保留接口：复杂外交博弈、多层军队作战 AI、全球金融策略、完整舆论操纵和跨代动态剧情生成。
